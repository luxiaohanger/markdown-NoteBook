## AD OSR OOD
AD、OSR、OOD的核心区别在于它们处理“未知”问题的侧重点不同：**AD（异常检测）** 关注数据点是否偏离“正常”；**OSR（开放集识别）** 关注分类时能否拒绝“未知类别”；**OOD（分布外检测）** 关注输入数据是否来自训练分布之外。

下面为您详细解释：

1.  **异常检测 anomaly detection**
    *   **核心目标**：在主要由“正常”样本构成的数据中，识别出罕见的、异常的或与预期模式显著不同的数据点。
    *   **关键思想**：学习“正常”的模式或边界，将不符合此模式的数据判定为异常。
    *   **典型场景**：工业质检（找出瑕疵品）、金融风控（检测欺诈交易）、网络入侵检测等。

2.  **开放集识别 open set recognition**
    *   **核心目标**：在分类任务中，模型不仅能正确分类已知的类别，还能识别出输入样本属于训练时未见过的“未知类别”，并将其拒绝。
    *   **关键思想**：在分类器的决策边界外，为“未知”留出空间。它通常需要对已知类别的置信度或可能性进行校准。
    *   **典型场景**：人脸识别（遇到数据库中没有的人）、物体识别（遇到没学过的物体类别）等。

3.  **分布外检测 Out-of-distribution**
    *   **核心目标**：判断输入数据是否与模型训练时所依赖的数据分布不同。如果输入是OOD样本，模型对其预测可能不可靠。
    *   **关键思想**：评估输入数据与训练数据分布的相似性。这比单纯的“异常”定义更一般化，关注的是整个数据分布的偏移。
    *   **典型场景**：评估模型在真实世界中的安全性与可靠性。例如，一个只在白天图片上训练的自动驾驶模型，需要检测夜晚图片作为OOD输入。

**简单比喻**：
*   **AD**：在一箱好苹果里找出烂苹果。
*   **OSR**：一个只能识别“苹果”和“香蕉”的系统，看到“橘子”时会说“不认识”。
*   **OOD**：一个用卡通动物图片训练的分类器，判断一张真实照片是否超出了它的“认知范围”。

**三者联系与总结**：
OSR和OOD检测的目标高度重合，常被一起讨论。AD可视为OOD检测的一种具体应用场景（将“异常”视为一种特殊的分布外数据）。下表概括了主要区别：

| 概念 | 核心问题 | 主要任务场景 | 关注层面 |
| :--- | :--- | :--- | :--- |
| **AD** | 这是不是异常/离群点？ | 单类或正常-异常二分类 | 数据点层面 |
| **OSR** | 这属于已知类别还是未知类别？ | 多分类，且包含“未知”类 | 类别层面 |
| **OOD** | 这个输入是否来自训练分布之外？ | 任何机器学习模型的安全性评估 | 数据分布层面 |

在实际研究和应用中，这些领域的界限有时比较模糊，方法也常互相借鉴。

## data argumentations
数据增强是一种在机器学习，特别是深度学习领域中，用于**人工扩充训练数据集**的技术。其核心思想是：通过对原始训练数据进行一系列**合理的、保持标签不变**的随机变换，来生成新的、多样化的训练样本，从而提升模型的泛化能力和鲁棒性。

### 一、核心目的与价值
1.  **解决数据稀缺问题**：在许多实际场景中，获取大量高质量的标注数据成本高昂。数据增强能“无中生有”，有效增加训练数据量。
2.  **防止模型过拟合**：如果训练数据有限或多样性不足，模型容易记住训练集中的噪声和特定模式，而在未见数据上表现不佳。增强后数据更具多样性，迫使模型学习更本质、更通用的特征。
3.  **提升模型鲁棒性**：通过模拟现实世界中可能遇到的各种变化（如光照变化、遮挡、视角变化、同义词替换等），让模型对这些干扰因素不敏感，从而更稳定。
4.  **引入噪声与正则化**：数据增强可以看作是一种在输入空间进行的正则化技术，类似于Dropout（在网络中引入噪声）或权重衰减，有助于模型平滑学习。

### 二、主要方法（按数据类型分类）

#### 1. 图像数据增强
这是应用最广泛的领域，方法主要分为**几何变换**和**像素变换**两大类。

*   **几何变换**：
    *   **旋转/翻转**：将图像随机旋转一定角度（如±30°），或进行水平/垂直翻转。适用于目标方向不固定的场景。
    *   **裁剪与缩放**：随机裁剪图像的一部分并缩放到固定尺寸。这能强迫模型不依赖于物体的绝对位置和完整出现。
    *   **平移/扭曲**：沿X或Y轴随机移动图像，或进行轻微的仿射/透视变换。
*   **像素变换**：
    *   **颜色抖动**：随机调整图像的亮度、对比度、饱和度和色调。
    *   **添加噪声**：向图像中添加高斯噪声、椒盐噪声等，提升模型抗噪能力。
    *   **模糊/锐化**：应用高斯模糊或锐化滤波器。
    *   **擦除**：随机将图像中的一块矩形区域置为0或随机值，模拟遮挡，迫使模型关注物体的多个部分。
*   **混合增强**：
    *   **MixUp/CutMix**：将两张图像及其标签以某种方式线性混合。例如，CutMix是将一张图的部分区域裁剪并粘贴到另一张图上，标签按面积比例混合。这种方法能鼓励模型学习更平滑的决策边界。

#### 2. 文本数据增强
*   **词汇层面**：
    *   **同义词替换**：随机选择非停用词，用其同义词替换。
    *   **随机插入/交换/删除**：随机插入一个词的同义词、随机交换相邻两个词、或随机删除一个词。
*   **句子层面**：
    *   **回译**：将句子翻译成另一种语言（如英文->法文），再翻译回原语言（法文->英文）。这种方法能生成语法正确但表述不同的句子。
    *   **文本生成模型**：使用预训练语言模型根据上下文生成新的、语义相似的句子。

#### 3. 音频数据增强
*   **时间扭曲**：轻微加快或放慢音频速度。
*   **音高变换**：改变音频的音调。
*   **添加噪声**：加入背景噪声（如白噪声、环境音）。
*   **时间掩蔽/频率掩蔽**：在频谱图上随机屏蔽一段连续的时间步或频率通道。

### 三、应用实践与要点
1.  **增强强度**：增强的强度需要仔细调节。过于轻微的增强可能效果不明显；过于激进的增强（如将猫的图片旋转180度）可能会改变其语义，导致标签错误，损害模型性能。
2.  **领域适应性**：增强方法必须符合数据的**领域知识**。例如，在手写数字识别中，垂直翻转“6”会变成“9”，导致标签错误；但在自然场景物体识别中，垂直翻转通常是安全的。
3.  **在线增强 vs 离线增强**：
    *   **在线增强**：在每个训练周期，数据加载时实时进行随机增强。这是最常用的方式，能产生近乎无限的数据变体。
    *   **离线增强**：预先生成增强后的数据集并保存。适用于增强方法非常耗时，或需要严格固定增强集的情况。
4.  **与模型训练的结合**：现代深度学习框架（如PyTorch的`torchvision.transforms`， TensorFlow的`tf.image`）都内置了方便的增强模块，可以轻松集成到数据加载管道中。

### 总结
数据增强是一种简单、高效且成本极低的提升模型性能的正则化技术。它通过扩展数据分布的覆盖范围，引导模型学习到更鲁棒和通用的特征，是深度学习工程师工具箱中不可或缺的工具。其成功应用的关键在于根据具体任务和数据特性，设计和组合恰当的增强策略。

## covariate shifts
### 核心定义
**协变量偏移**是分布外泛化问题中最常见的一种分布偏移。它特指以下情况：
*   **输入数据的边缘分布 \(P(X)\) 发生变化**，即测试环境（或部署环境）中输入的统计特性与训练环境不同。
*   **给定输入后输出的条件分布 \(P(Y|X)\) 保持不变**，即输入与输出之间的因果关系或判别规则没有改变。

**简单比喻**：你训练了一个模型来根据照片判断动物是猫还是狗。训练数据全部是在晴天室内拍摄的清晰照片（分布A）。现在，你将模型用于分析阴天户外拍摄的模糊照片（分布B）。这里，照片的“风格”（光照、清晰度、背景）发生了巨大变化，这就是**协变量（X）的偏移**。但“猫”和“狗”本身的定义（给定一张清晰照片，如何判断它是猫还是狗的逻辑）并没有改变，这就是**条件分布 \(P(Y|X)\) 不变**。

### 为什么这是一个严重问题？
即使真实规律 \(P(Y|X)\) 没变，协变量偏移也会导致模型性能严重下降。因为现代机器学习模型（尤其是深度学习）很容易学习到数据中的**虚假相关性**或**表面统计特征**，而非真正的因果机制。

**经典例子**：
*   **背景作为虚假特征**：训练一个“牛”分类器，训练数据中的牛总是出现在草地上。模型可能学会通过识别“草地”来预测“牛”，而不是牛本身的形态特征。当测试图片中的牛出现在沙滩上时（协变量偏移），模型就会失效。
*   **医疗诊断**：基于某家医院特定设备（如特定品牌的MRI机器）采集的数据训练疾病诊断模型。当模型部署到使用不同品牌设备的另一家医院时，图像风格（对比度、噪声）的差异（协变量偏移）可能导致诊断准确性下降。

### 与其他类型分布偏移的对比
为了更清晰理解，我们将其与另外两种主要偏移进行对比：

| 偏移类型 | 定义 | 通俗例子 |
| :--- | :--- | :--- |
| **协变量偏移** | \(P(X)\) 变， \(P(Y\|X)\) 不变 | 训练：晴天清晰猫图；测试：阴天模糊猫图。判断猫的规则没变，但图片风格变了。 |
| **标签偏移** | \(P(Y)\) 变， \(P(X\|Y)\) 不变 | 训练：猫狗数量各半；测试：90%是猫，10%是狗。每种动物长得样子（\(P(X\|Y)\)）没变，但它们的出现频率（先验\(P(Y)\)）变了。 |
| **概念漂移** | \(P(Y\|X)\) 变 | 垃圾邮件过滤：过去“免费”一词是强垃圾信号，现在正常邮件也常用。词（X）和垃圾邮件（Y）之间的关系发生了变化。 |

### 如何应对协变量偏移？
研究领域提出了多种方法来提升模型对协变量偏移的鲁棒性：

1.  **领域适应**：假设我们拥有**少量有标签**的目标域（测试分布）数据。目标是利用源域（训练分布）的大量数据和目标域的少量数据，让模型适应目标域。常用方法包括：
    *   **特征对齐**：通过领域对抗训练，让模型学习到的特征无法被区分是来自源域还是目标域，从而提取域不变特征。
    *   **自训练/伪标签**：用模型对目标域无标签数据生成高置信度伪标签，加入训练。

2.  **领域泛化**：一个更严格且现实的设定——在训练时**完全无法访问**目标域数据。目标是在多个不同的源域上训练，让模型能泛化到未知的新域。方法包括：
    *   **领域增强**：对源域数据进行激进的数据增强，模拟可能遇到的分布变化（可视为你上一个问题中数据增强的高级应用）。
    *   **不变性学习**：强制模型学习在所有源域上都保持稳定的因果特征，丢弃那些随域变化的虚假特征（如背景风格）。

3.  **重要性加权**：一种经典统计方法。核心思想是：由于 \(P(Y\|X)\) 不变，测试样本在训练数据中只是“权重”不对。我们可以重新加权训练样本，让加权后的输入分布逼近测试分布。但估计这些权重本身通常需要一些目标域数据。

4.  **模型设计与正则化**：
    *   使用更简单、正则化更强的模型，避免过度拟合训练域的特有虚假特征。
    *   在损失函数中加入旨在提升泛化能力的正则项。

### 总结
协变量偏移揭示了机器学习模型从经验数据中学习时的一个根本性脆弱点：它们倾向于拟合最容易利用的表层统计规律。理解协变量偏移是构建在现实世界中稳定、可靠AI系统的关键第一步。当前的研究前沿（如领域泛化、不变性学习、因果表示学习）都致力于从根本上解决这一问题，使模型能够抓住“事物之所以如此”的本质原因，而非表面的相关现象。

An ideal system should be robust to the non-semantic covariate shift （OOD generalization）while being able to identify semantic shift (OOD detection).

## OOD Generalization & OOD Detection
OOD泛化（OOD Generalization）与OOD检测（OOD Detection）是机器学习中处理分布外（Out-of-Distribution）数据的两种不同范式，核心区别在于目标与行为方式。

简单来说：
*   **OOD泛化**的目标是**让模型“学会应对”未知分布**，在训练时利用OOD信息提升模型在多种未知环境下的直接预测性能。
*   **OOD检测**的目标是**让模型“识别出”未知分布**，在部署时判断输入是否属于未知类别或分布，并决定是否拒绝预测。

为了更清晰地展示，它们的核心区别如下：

| 对比维度 | OOD泛化 (OOD Generalization) | OOD检测 (OOD Detection) |
| :--- | :--- | :--- |
| **核心目标** | 提升模型在**未见过的测试分布**上的**预测性能**。 | 识别出输入数据是否来自**训练分布之外**，并进行**二元判断**（是/否）。 |
| **训练数据** | 通常需要包含来自**多个不同分布**的数据（如不同环境、领域），旨在学习不变特征。 | 通常**仅使用**来自**单一训练分布**的数据进行训练。 |
| **部署行为** | 对**所有输入**直接给出预测结果，目标是该结果在OOD数据上也尽可能准确。 | 对输入先进行**判别**：若判定为ID（分布内）则给出预测；若判定为OOD则**拒绝预测**或触发警报。 |
| **典型场景** | 模型需要部署到多个未知环境，如在不同医院使用的医疗诊断模型、在不同城市运行的自动驾驶系统。 | 需要保证安全或可靠性的场景，如人脸识别闸机拒绝陌生人、工业质检系统标记异常产品。 |

**两者的联系与选择**
它们是互补的技术。一个强大的系统可以同时具备OOD检测能力（过滤危险未知输入）和OOD泛化能力（对可接受的未知变化保持鲁棒）。选择哪种方法取决于应用需求：若无法承受OOD输入上的错误预测，则需OOD检测；若必须对所有输入给出预测且希望性能稳定，则需研究OOD泛化。

## space
神经网络中的“空间”是对数据表示的一种抽象理解。简单来说，它们都是将数据（如图片、文字）映射为一系列数字（向量）后，这些向量所存在的数学空间。

核心理解如下：

**1. 特征空间**
这是最广义的概念。**神经网络任何中间层的输出，都可以称为该层学习到的“特征”，这些特征向量所在的范围就是特征空间。** 它像是网络从数据中提取的“观察视角”。
*   **是什么**：数据经过网络部分层变换后的表示。
*   **特点**：维度可能很高，且包含大量信息（有些可能对当前任务无用或冗余）。
*   **作用**：用于可视化理解网络每一层在“看”什么，是模型内部信息的直接体现。

**2. 嵌入空间**
这是一种特殊的、经过精心优化的特征空间。**它是为了下游任务（如检索、比较）而学习到的低维、稠密、有意义的特征空间。** 在这个空间里，语义相似的数据点距离很近。
*   **是什么**：通常是网络倒数第二层等最终输出的特征空间，专为相似性度量设计。
*   **特点**：维度相对较低，向量稠密，几何距离直接反映语义相似度。
*   **作用**：是人脸识别、推荐系统、语义搜索等任务的核心。例如，所有“猫”的图片在嵌入空间中会聚集在一起。

**简单比喻**：
*   **特征空间** 像是一幅画的**各种细节观察报告**（色彩报告、笔触报告、构图报告）。报告维度高，信息全。
*   **嵌入空间** 像是根据这些报告，最终给出的**一份精简的艺术流派鉴定书**（“印象派，与莫奈风格相似度85%”）。它维度低，且直接服务于“分类和比较”这个任务。

**其他常见空间**：
*   **输入空间**：原始数据（如像素矩阵、单词序列）所在的空间，通常难以直接处理。
*   **潜在空间**：常指生成模型（如VAE, GAN）中，那个压缩了数据关键特征的隐藏空间。在这个空间里操作向量，可以平滑地生成新数据。**嵌入空间可以看作是一种服务于判别任务的潜在空间。**

**总结关系**：
**嵌入空间是特征空间的一种**，是经过优化、用于语义匹配的特例。而**潜在空间**强调了其隐藏、压缩的特性，在生成模型中尤为关键。神经网络的学习过程，本质上就是将数据从**输入空间**，通过多层变换，映射到有意义的**特征空间**，并最终形成一个有利于解决特定任务（如分类的决策边界空间，或检索的嵌入空间）的表示空间。

## Contrastive Learning

对比学习是一种通过**让模型学会区分数据的“相似”与“不相似”** 来学习高质量数据表示的机器学习方法。其核心目标是构建一个**表示空间**，在此空间中，语义相似的样本彼此靠近，不相似的样本彼此远离。

### **一、两大范式：根本区别在于“相似性”的定义**
对比学习根据定义“相似性”的信号来源，分为两大范式：

| 维度 | **无监督/自监督对比学习** | **监督对比学习** |
| :--- | :--- | :--- |
| **监督信号来源** | **数据自身**（通过预设的变换规则） | **人工标注的标签** |
| **正样本定义** | **同一原始样本的不同增强视图**（如一张图片经过裁剪、变色后的两个版本）。核心假设：**同一事物的不同视角在语义上相似**。 | **共享同一人工标签的所有样本**。核心事实：**被标记为同一类别的样本在语义上相似**。 |
| **负样本定义** | 批次中或记忆库中**其他任意样本**（视为不相似）。 | **拥有不同人工标签的所有样本**（视为不相似）。 |
| **核心目标** | 学习对**各种数据增强保持不变**的、通用的特征表示。 | 学习具有**高度类别判别性**的特征表示。 |
| **经典方法** | SimCLR, MoCo | Supervised Contrastive Loss |

### **二、通用工作流程与关键组件**
尽管两种范式的输入不同，但它们共享一个相似的框架：

1.  **样本对构建**：根据上述规则，为每个样本构建正样本对和负样本集合。
2.  **特征编码**：通过一个编码器网络将每个样本映射为一个低维特征向量。
3.  **相似度计算**：在特征空间中计算样本对之间的相似度。
4.  **损失优化**：使用对比损失函数进行优化。

**关键组件详解：**
*   **编码器**：通常是神经网络，负责提取特征。
*   **投影头**：一个小型网络，将编码后的特征映射到更适合进行对比度度量的空间。
*   **损失函数**：
    *   **InfoNCE Loss**：无监督对比学习的基石。它促使模型识别出哪个样本是当前样本的正视图，而不是大量的负样本。
    *   **Supervised Contrastive Loss**：监督对比学习的核心。它将每个样本与同一批次内所有同类样本拉近，与所有异类样本推远。

### **三、核心优势与价值**
*   **无监督对比学习**：
    *   **高效利用海量无标签数据**，是构建通用基础模型的关键。
    *   学习到的表示**可迁移性极强**，能显著提升下游任务性能。
*   **监督对比学习**：
    *   **超越传统交叉熵损失**，能学习到**类内更紧凑、类间更分离**的特征分布，提升模型的判别能力和鲁棒性。

### **总结**
对比学习是一种强大的**表示学习**范式。**无监督对比学习**从数据自身寻找规律，是通往通用人工智能的重要路径；**监督对比学习**则利用标签信息，旨在学习更优越的判别性特征。两者共同的思想是“通过对比进行学习”，已成为现代深度学习不可或缺的组成部分。

## CE Loss & SupCon Loss
Softmax交叉熵损失（CE Loss）和对比损失（Contrastive Loss）的核心区别在于：**CE Loss用于分类任务，目标是让模型输出正确的类别标签；而对比损失用于度量学习，目标是直接学习样本特征的相似性关系，让相似样本的特征靠近，不相似样本的特征远离。**

以下是两者的详细对比：

| 维度 | Softmax 交叉熵损失 (CE Loss) | 对比损失 (Contrastive Loss) |
| :--- | :--- | :--- |
| **核心目标** | 最大化**单个样本**被正确分类的概率。 | 学习一种**特征表示**，使得相似样本对的特征距离小，不相似样本对的特征距离大。 |
| **输入形式** | 单个样本的特征向量和其对应的**类别标签**。 | **样本对**（正样本对或负样本对）和**关系标签**（相似/不相似）。 |
| **输出/优化对象** | 优化模型最后一层（分类层）的权重，以输出正确的类别概率分布。 | 优化模型的特征提取器（编码器），以产生具有判别性的特征向量。 |
| **主要应用场景** | 标准的**有监督分类**任务（如图像分类、文本分类）。 | **度量学习**、**人脸识别**、**图像检索**、**自监督学习**（如SimCLR）等需要比较样本相似度的任务。 |
| **关系处理方式** | **隐式地**通过共享的分类器权重来建立类别间的区分关系。 | **显式地**通过定义正负样本对，直接约束特征空间中的距离关系。 |
| **优点** | 目标直接，在分类任务上非常有效且高效。 | 能学习到更通用、更具迁移性的特征表示，对未知类别更友好。 |
| **缺点** | 学到的特征可能过于依赖特定分类任务，泛化性可能不足；难以处理类别数极多或动态变化的场景。 | 构造有效的正负样本对是关键难点；训练计算成本通常更高（需要成对或成批对比）。 |

**简单来说**：你可以把CE Loss看作一个“**裁判**”，它只关心“这张图片是不是狗”。而对比损失是一个“**策展人**”，它关心的是“这两张图片（或句子）在内容上有多像”，并据此整理整个特征画廊的布局。在现代实践中，这两种思想也常被结合使用，例如在分类损失基础上增加一个对比损失作为正则项，以同时获得好的分类性能和强的特征表示。

## Mahalanobis distance
**马氏距离是一种衡量数据点与一个数据分布之间“真实”距离的度量方法，它考虑了数据各维度之间的相关性和尺度差异。**

与直接计算直线距离的**欧氏距离**不同，马氏距离通过数据的协方差结构进行校正，相当于在计算距离前，先将数据旋转和缩放成一个“标准化”的球形分布。

**核心思想与公式**：
对于一个数据点 \( \mathbf{x} \) 和一个均值为 \( \mathbf{\mu} \)、协方差矩阵为 \( \mathbf{\Sigma} \) 的数据分布，其马氏距离 \( D_M \) 定义为：
\[
D_M(\mathbf{x}) = \sqrt{(\mathbf{x} - \mathbf{\mu})^T \mathbf{\Sigma}^{-1} (\mathbf{x} - \mathbf{\mu})}
\]
*   \( \mathbf{\Sigma}^{-1} \) 是协方差矩阵的逆，它起到了“校正”作用。

**关键特性与优势**：
1.  **尺度无关性**：它自动处理不同特征具有不同量纲和方差的问题。例如，一个维度是“公里”，另一个是“米”，马氏距离能公平对待。
2.  **考虑相关性**：如果两个特征高度相关，马氏距离会减弱它们共同贡献的信息，避免重复计算。这是它最核心的优势。
3.  **与数据分布直接相关**：在多元正态分布的假设下，数据点到一个分布的马氏距离的平方服从卡方分布，这为统计检验（如异常检测）提供了理论依据。

**一个直观比喻**：
想象一片倾斜的椭圆形沙滩（代表二维数据的分布）。要判断一个脚印（数据点）离人群中心（均值点）有多“远”：
*   **欧氏距离**：像在空中用直尺测量直线距离，会错误地认为沿着椭圆长轴方向很远的一个点（其实仍在沙滩分布内）比短轴方向很近的一个点（可能已在沙滩外）更远。
*   **马氏距离**：它会先“摆正”这片沙滩，把它变成一个正圆形，然后再测量距离。这样，沿着长轴方向的距离会被“压缩”，而沿着短轴方向的距离会被“拉伸”，从而得出符合数据真实分布的“有效”距离。

**主要应用场景**：
*   **异常检测**：计算样本到正常数据分布的马氏距离，距离过大的即可判为异常。
*   **线性判别分析**：在分类中，用于计算样本到各类别中心的距离，并考虑特征的共同方差结构。
*   **多元统计分析**：用于假设检验和置信区域的构建。

**简单总结**：马氏距离是**特征空间**或**嵌入空间**中一种更科学的距离度量。当你的数据特征维度之间存在不同的尺度或相关性时，使用它比欧氏距离更能反映真实的样本相似性或差异性。

## Sun 等 - 2022 - Out-of-Distribution Detection with Deep Nearest Neighbors
### non-parametric
根据论文内容，本文提出的基于深度最近邻（KNN）的OOD检测方法被称为“non-parametric”（非参数）方法，主要是因为它与先前主流的“parametric”（参数化）方法存在根本性的区别。具体依据如下：

1.  **核心区别：是否对数据分布做出强假设**
    *   **先前参数化方法（如Mahalanobis距离）**：论文指出，以Lee等人（2018）和SSD+（Sehwag等，2021）为代表的方法，**假设**内分布（ID）数据的特征嵌入空间服从**类条件多元高斯分布**（a class-conditional multivariate Gaussian distribution）。这是一种强烈的、预设的分布假设。论文第1节通过Henze-Zirkler检验验证了学习到的特征向量并不服从正态分布，表明这种假设可能不成立，从而限制了方法的灵活性和通用性。
    *   **本文的非参数KNN方法**：与之相对，KNN方法**不**对底层特征空间的分布形式做出任何假设。论文在多处强调了这一点，例如：
        *   摘要：“our method does not impose any distributional assumption”。
        *   第1节引言：“the non-parametric approach does not impose any distributional assumption about the underlying feature space”。
        *   第3节：“KNN offers compelling advantages of being: (1) distributional assumption free”。
        该方法直接基于训练数据特征点之间的距离（k-th nearest neighbor distance）进行决策，而不需要估计或拟合任何参数化的概率分布模型（如高斯分布的均值和协方差）。

2.  **方法实现体现了非参数特性**
    *   KNN方法的核心操作是计算测试样本特征与所有训练样本特征之间的欧氏距离，并取第k小的距离作为得分。这个决策边界**完全由数据本身决定**，随着数据量和数据分布的变化，决策边界可以自适应地形成复杂的形状，而不受限于某个参数化模型（如高斯）所定义的简单几何形状（如超椭球）。论文在第6节的理论分析中也将其描述为“非参数水平集估计”。

3.  **与参数化方法对比的优势**
    *   论文将KNN与基于**马氏距离（Mahalanobis distance）** 的方法（一种典型的参数化方法）进行了直接对比。马氏距离的计算依赖于对每个类别的特征均值（类中心）和协方差矩阵的估计，并且需要求协方差矩阵的逆。论文指出，这在数据维度高时可能**数值不稳定**，且计算成本较高。
    *   相比之下，KNN方法**无需计算协方差矩阵或其逆**，仅需计算距离，并通过高效的近似最近邻搜索库（如Faiss）实现，即使对于大规模数据集（如ImageNet）也能保持较快的推理速度。论文在实验结果（如表4）中显示，KNN在保持高性能的同时，推理时间与大部分基线方法相当。

**总结**：论文将其方法称为“non-parametric”，是因为它完全摒弃了对内分布数据特征空间做出特定参数化分布（如高斯）假设的范式，转而采用一种完全由数据驱动、基于距离的最近邻准则。这使得该方法具有更强的灵活性、通用性和易用性。

### threshold
根据论文内容，用于判断样本是否为OOD（分布外）的阈值 λ 是通过以下方式确定的：

1.  **阈值的设定目标**：该阈值被设定为能够确保**高比例的内分布（ID）数据**被正确分类。论文中明确指出：“The threshold λ is typically chosen so that a high fraction of ID data (e.g., 95%) is correctly classified。”（第3节）。这意味着，在选择阈值时，目标是让95%的ID样本的KNN距离得分高于该阈值，从而被模型判断为ID。

2.  **阈值的估计依据**：阈值**仅依赖于ID数据**，而不需要任何OOD数据的信息。论文在描述其方法优势时特别强调了这一点：“OOD-agnostic (i.e., the distance threshold is estimated on the ID data only, and does not rely on information of unknown data)”（第3节）。这与许多需要接触OOD样本进行训练或调参的方法形成对比。

3.  **具体的实现方式**：虽然论文没有给出选择阈值的详细代码步骤，但其遵循的通用做法是：在模型训练完成后，使用一个**独立的ID验证集**（来自训练数据的分割）来计算所有验证样本的KNN距离得分。然后，根据预设的ID样本召回率目标（例如95%），在该验证集的得分分布上选择对应的百分位数作为阈值 λ。例如，选择使得95%的验证集样本得分都高于它的那个值作为阈值。

4.  **在评估中的体现**：这种阈值选择方式直接对应于论文中报告的核心评估指标之一：**FPR95**（即当ID样本的真阳性率（TPR）被固定为95%时，OOD样本的假阳性率）。这表明在评估时，阈值正是在ID测试集上按照95%的TPR来校准的。

**总结**：阈值 λ 是在一个纯ID数据的验证集上，通过调整并固定ID样本的**正确接受率**（例如95%）来确定的。它是一个数据驱动的、无需OOD先验知识的超参数。
