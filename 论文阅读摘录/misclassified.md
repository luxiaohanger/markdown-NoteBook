## Misclassified & OOD
简单来说：
*   **Misclassified：模型“认错了”** —— 它认识这个东西，但判断错了类别。
*   **Out-of-Distribution：模型“没见过”** —— 这个东西完全在它的认知范围之外。

我们可以通过一个生动的类比来理解：

> **想象你在训练一个“动物分类器”，只教它认识猫和狗。**
>
> *   **Misclassified：** 你给它看一张**哈士奇**的照片，模型信心十足地输出“**猫**”。模型认识狗（因为它训练时见过其他狗），但它把这只具体的狗错误地归类了。这是**模型能力或决策边界**的问题。
> *   **Out-of-Distribution：** 你给它看一张**汽车**的照片。模型可能会输出“狗”（概率0.6）和“猫”（概率0.4）。但这个预测是**无意义**的，因为汽车根本不是它训练过的类别。模型在“胡说八道”，因为它被迫在自己知识范围外做选择。这是**数据定义域**的问题。

下面是详细的对比表格：

| 特性 | **误分类** | **分布外** |
| :--- | :--- | :--- |
| **核心定义** | 模型在**已知分布内**做出了错误的预测。 | 输入数据来自模型训练时**从未见过**的分布。 |
| **类比** | 学生考试，题目都在大纲内，但他答错了。 | 考试题目完全超出了教学大纲的范围。 |
| **问题根源** | 1. 模型能力不足（太简单）<br>2. 训练数据噪声或标注错误<br>3. 特征混淆不清<br>4. 模型过拟合/欠拟合 | 1. 训练数据覆盖不全，现实世界出现新情况<br>2. 测试环境与训练环境不同（如白天训练，夜晚测试）<br>3. 对抗性攻击或异常输入 |
| **模型“视角”** | 模型认为自己“认识”这个样本，并给出了一个（错误的）高置信度预测。 | 模型感到“困惑”，因为它面对的是陌生模式。它可能**仍会给出一个高置信度的错误预测**，这是最危险的地方。 |
| **解决方案** | 1. 使用更复杂的模型<br>2. 提供更多/更干净的训练数据<br>3. 数据增强<br>4. 调整模型超参数 | 1. 收集更多样化的训练数据<br>2. 使用**OOD检测技术**（在预测前先判断输入是否异常）<br>3. 使用不确定性估计方法<br>4. 在部署中建立人工审核流程 |
| **举例** | 训练一个数字识别器（0-9）：<br>- 把**手写的“9”** 识别为“**4**”。 | 训练一个数字识别器（0-9）：<br>- 输入一张**字母“A”** 的图片。<br>- 输入一张**猫**的图片。<br>- 输入一张**背景完全变黑**的数字图片。 |

### 关键总结与联系

1.  **层次不同**：**OOD是更根本、更前置的问题**。当模型遇到OOD样本时，讨论“分类是否正确”已经失去了意义，因为正确的类别根本不在选项里。而Misclassified是在“可以分类”这个前提下的性能问题。
2.  **危险程度**：**OOD通常更危险**。一个Misclassified的狗被认成猫，可能只是精度下降。但一个OOD的“自动驾驶车辆识别器”看到从未见过的路障，可能会以极高置信度将其归类为“天空”或“道路”，导致严重事故。
3.  **诊断**：在实际应用中，如果模型在测试集上表现突然变差，首先要排查是否是**数据分布发生了变化**（出现了OOD数据），而不是简单地认为模型分类能力（Misclassified）下降了。

# Hendrycks和Gimpel - 2017

- 正类和负类：也即一对相对的概念，具体可以是分类正确/错误、分布内/外，其中要指定正类的定义

Hendrycks和Gimpel - 2018 中采用softmax层最大值作为分数，核心观点是正确分类/分布内样本的分数应当统计上高于错误分类/分布外样本，但是分数本身不能作为置信度，因为错误分类的样本可能也具有较高的最大分数，这是由于softmax指数性质决定的

为了比较正确/错误分类的softmax最大分数，我们划定一个判定阈值，规定阈值两侧的样本分别划分为正类和负类，而阈值的不同就会造成划分的效果不同

对于某个阈值，我们定义：

- 真正类：TP true positive 本身是正类，被正确划分为正类
- 假正类：FP false positive 本身是负类，被错误划分为正类
- 真负类：TN true negative 本身是负类，被正确划分为负类
- 假负类：FN false negative 本身是正类，被错误划分为负类

为了衡量阈值的划分效果，我们定义：
- 真正例率tpr（召回率）：TP/（TP + FN）所有正类中被划分为正类的概率
- 假正例率fpr：FP/（FP + TN）所有负类中被划分为正类的概率
- 精确率：TP（TP + FP）所有被划分为正类的样本中，真正是正类的概率

### FPR@TPR95
当tpr = 0.95时的fpr，越低越好

### AUROC   AUPR
因此我们引出了一种与阈值无关的方法，通过遍历所有可能的阈值，每一个阈值都会产生一对（FPR, TPR）值。以FPR为横轴，TPR为纵轴，将所有点连接起来，就得到了ROC曲线。

计算AUROC：AUROC​ 即是这条ROC曲线下的面积。论文进一步解释，AUROC值可以直观理解为“一个正例样本的检测分数高于一个负例样本的概率”。因此，随机猜测对应AUROC为50%，完美检测器对应100%。

AUROC 避开了阈值选择的问题，与之类似的是精确率-召回率曲线下的面积（AUPR），有时人们认为它更具信息量（Manning & Schütze，1999）。这是因为当正类和负类的基础率差异很大时，AUROC 并非理想选择，而 AUPR 能够对这些不同的正负基础率进行调整。因此，AUPR 是我们的第二个评估指标。PR 曲线将精确率（tp/(tp+fp)）和召回率（tp/(tp + fn)）相互绘制。基线检测器的 AUPR 大约等于其精确率（Saito & Rehmsmeier，2015），而“完美”的分类器的 AUPR 为 100%。

## Grid-search
（网格搜索）是一种系统性的超参数调优方法。它通过为每个需要调优的超参数预先设定一组候选值，然后遍历所有这些值组合成的“网格”，来寻找使模型性能最优的参数组合。

简单来说，就像在一个多维表格中，对每一个格子（即每一种参数组合）都训练并评估一次模型，最终选出得分最高的那个格子对应的参数。

它的核心步骤是：
1.  **定义参数网格**：确定要调优的参数（如学习率、树的最大深度）以及每个参数的候选值列表。
2.  **遍历与训练**：使用交叉验证等方法，对每一种参数组合都训练一个模型。
3.  **评估与选择**：根据预设的评估指标（如准确率、F1分数）对所有模型进行评分，最终选择性能最好的参数组合。

**主要特点：**
*   **优点**：方法简单直观，由于是穷举搜索，只要网格足够细密，就一定能找到给定范围内的最优解。
*   **缺点**：计算成本非常高，所需时间随参数数量呈指数级增长。因此，它通常用于待调参数不多、候选值范围较明确的场景。

在实际应用中，对于参数较多的复杂模型，人们常会先使用网格搜索进行粗调，再使用**随机搜索（RandomizedSearch）** 或更高级的**贝叶斯优化（Bayesian Optimization）** 等方法进行更高效的精细调优。

## SVM
支持向量机（Support Vector Machine， SVM）是一种强大的监督学习算法，主要用于**分类**任务，也可用于回归和异常检测。它的核心思想是寻找一个最优的决策边界（超平面），以最大化不同类别样本之间的间隔。

我们可以分三步来理解其原理：

**1. 最优超平面与“间隔”最大化**
想象在二维平面上有两类完全可分的点。SVM的目标不是简单找一条线分开它们，而是找到一条“最好”的线。这条“最好”的线被定义为能使两侧**距离该线最近的样本点**（这些点被称为**支持向量**）最远的线。这个距离被称为“间隔”。最大化间隔意味着决策边界具有最强的泛化能力，对未知数据的分类最鲁棒。

**2. 处理线性不可分：“软间隔”**
现实中数据常有噪声或重叠，无法完美线性分割。SVM引入了“软间隔”概念，允许一些样本点落在间隔之内甚至被错误分类。通过一个惩罚参数 **C** 来控制这种容忍度：C值越大，对误分类的惩罚越重，模型越倾向于将所有训练样本分类正确（可能导致过拟合）；C值越小，则允许更多的误分类以换取更大的间隔和更好的泛化能力。

**3. 升维解决非线性问题：“核技巧”**
对于在原始空间中完全无法用线性平面分割的数据（例如同心圆），SVM使用了一个巧妙的“核技巧”。其核心是将数据通过一个函数映射到一个更高维的特征空间，使得在高维空间中数据变得线性可分。而计算时，并不需要真正进行复杂的高维映射，只需直接计算原始空间中的某个函数（即**核函数**），其效果等价于在高维空间中进行点积运算。常用的核函数有：
*   **线性核**：用于线性可分情况。
*   **多项式核**
*   **径向基函数核（RBF/Gaussian）**：最常用、最强大的核函数，通过调节参数 γ 控制单个样本的影响范围。



| 特点 | 说明 |
| :--- | :--- |
| **核心优势** | 基于间隔最大化理论，泛化能力强，尤其在中小规模、高维数据上表现优异。 |
| **关键概念** | **支持向量**：决定最终模型的关键样本，通常只占训练数据的一小部分。<br>**核技巧**：使SVM能有效处理复杂的非线性分类问题。 |
| **主要参数** | **C（惩罚系数）**：控制模型对误分类的容忍度。<br>**核函数类型及参数**（如RBF核的γ）：决定数据映射到高维空间的复杂程度。 |
| **适用场景** | 图像分类、文本分类、生物信息学等特征维度较高的领域。 |
| **注意事项** | 当特征数量远大于样本数量时，性能可能下降。对于大规模数据集，训练速度可能较慢。需要仔细调优C和核参数。 |

简单来说，SVM就像一个在构建城墙的建筑师，它的目标是让这座分隔两个王国的城墙（决策边界）尽可能远离双方最近的哨所（支持向量），并且拥有应对复杂地形（核函数）和局部冲突（软间隔）的灵活策略，以保障长久的稳定（强泛化能力）。

## FGSM
FGSM（Fast Gradient Sign Method，快速梯度符号法）是一种生成**对抗样本**的经典方法。对抗样本是指对原始输入（如图像）添加精心构造的微小扰动后形成的样本，该样本能“欺骗”深度学习模型，使其做出高置信度的错误预测。

### FGSM的核心原理

FGSM基于一个关键观察：深度学习模型的线性性质在高维空间中会使其对微小的扰动非常脆弱。其核心思想是**沿着使模型损失函数增大的方向（即梯度方向），对原始输入添加一个微小扰动**，从而最大化模型的预测错误。

**具体步骤：**
1.  **前向传播**：将原始干净图像输入模型，得到预测结果并计算损失（例如交叉熵损失）。
2.  **计算梯度**：计算损失函数**相对于输入图像像素**的梯度。这个梯度指明了“如何微调每个像素值，才能让模型的损失增加得最快”。
3.  **生成扰动**：取该梯度的**符号**（即每个像素梯度值的正负号，+1或-1），然后乘以一个很小的扰动幅度 **ε**。扰动公式为：
    `扰动 = ε * sign(∇x J(θ, x, y))`
    其中，`∇x J` 是损失函数对输入x的梯度，`sign`是符号函数，`ε`是控制扰动大小的参数。
4.  **生成对抗样本**：将生成的扰动加到原始图像上：
    `对抗样本 = 原始图像 + 扰动`

### 关键特点与影响

*   **白盒攻击**：FGSM是一种“白盒攻击”，需要完全了解目标模型的结构和参数，以便计算梯度。
*   **快速高效**：只需一次梯度计算和一次前向传播，速度极快。
*   **可视性与不可视性**：通过控制ε的大小，可以平衡攻击的成功率和扰动的可视性。ε很小时，扰动几乎人眼不可见，但依然能误导模型。
*   **核心目的**：
    *   **暴露模型脆弱性**：揭示模型并非真正“理解”数据，其决策边界在高维空间中存在不稳定性。
    *   **评估模型鲁棒性**：成为衡量模型对抗攻击能力的基准测试工具。
    *   **推动对抗训练**：FGSM生成的对抗样本可用于训练模型（即“对抗训练”），这是提升模型鲁棒性的最有效方法之一。

**后续发展**：FGSM是单步攻击方法。在其基础上，发展出了更强大、更隐蔽的**迭代式攻击**，如PGD（Projected Gradient Descent），它通过多次、小步长的FGSM攻击来生成对抗样本。

简单来说，FGSM就像是用一个精准的“小撬棍”，沿着模型最敏感的方向轻轻一撬，就能让这个看似强大的模型做出完全错误的判断。它深刻影响了机器学习安全领域的研究。

# Yan和Weng - 2025 - RAT Boosting Misclassification Detection Ability without Extra Data

在论文第4节中，作者首先介绍了标准对抗训练（Adversarial Training, AT）和反向对抗训练（Reversed AT），并解释了为什么简单地应用它们不能提升误分类检测（MisD）性能，从而引出了半径感知训练（RAT）的设计。

以下是根据论文第4节内容的解释：

1.  **定义与目标**：
    *   论文中的 **r** 指的是**鲁棒半径**，其定义为改变模型预测所需的最小扰动（公式(2)）。直观上，r 小的样本更靠近决策边界，模型对其预测的信心更低。
    *   论文第4节的目标是：**增大正确分类样本的 r（使其远离边界），同时减小误分类样本的 r（使其靠近边界）**，从而让模型更容易通过 r 的大小来区分对错。

2.  **标准对抗训练（AT）及其对 r 的影响**：
    *   标准AT的目标是：`min_Θ max_||x‘ - x||_∞ ≤ ε l(Θ; x‘)`。
    *   这里的 **内层最大化（max）** 是为了找到在扰动预算 ε 内能使损失最大的对抗样本。**训练模型去最小化这个“最坏情况”的损失**。
    *   这种训练迫使模型**对所有输入（无论对错）的预测在 small perturbation 下都保持稳定**。其效果是**普遍增大了训练样本的鲁棒半径 r**（如表1所示，AT 行的 “Radius (correct)” 和 “Radius (wrong)” 都比 Standard 行大）。
    *   然而，由于正确和错误样本的 r 都被增大了，它们之间的**相对差距**并没有明显改善，因此误分类检测性能（AUROC）没有提升。

3.  **反向对抗训练（Reversed AT）及其对 r 的影响**：
    *   反向AT的目标是：`min_Θ min_||x‘ - x||_∞ ≤ ε l(Θ; x‘)`。
    *   这里的 **内层最小化（min）** 是为了找到在扰动预算 ε 内能使损失最小的样本。**训练模型使得轻微扰动后的输入损失更小**。
    *   这相当于允许模型在输入有微小扰动时“犯点小错”，只要它离正确的决策区域不远。其效果是**普遍减小了训练样本的鲁棒半径 r**（如表1所示，ReverseAT 行的 r 比 Standard 行小）。
    *   同样，因为两类样本的 r 都被减小了，它们的区分度（AUROC）也没有改善。

4.  **半径感知训练（RAT）如何组合二者以改变 r**：
    *   RAT 的核心思想是针对**不同类别的样本**，在训练中应用**不同的内层优化目标**，从而**差异化地调节它们的 r**。
    *   其训练目标为（公式(11)）：
        *   对于**训练中误分类**的样本（`ŷ(x) ≠ y`）：应用 **`min_||x‘ - x||_∞ ≤ ε l(Θ; x‘)`** 。这与反向AT一致，目的是**减小**这些错误样本的 r，将它们“拉向”决策边界。
        *   对于**训练中正确分类**的样本（`ŷ(x) = y`）：应用 **`max_||x‘ - x||_∞ ≤ ε l(Θ; x‘)`** 。这与标准AT一致，目的是**增大**这些正确样本的 r，将它们“推离”决策边界。
    *   通过这种在训练过程中**有区别地（对不同样本分别采用 min 或 max 目标）进行优化**，RAT 成功实现了**增大正确样本 r 与减小错误样本 r 的同步进行**，从而扩大了两类样本在鲁棒半径 r 上的差距。这正是 RAT 能提升模型误分类检测能力的根本原因。

**总结**：
改变内层优化目标是 **`max`** 还是 **`min`**，直接决定了训练是迫使样本对抗扰动更鲁棒（增大r），还是允许样本对扰动更敏感（减小r）。RAT 方法通过根据样本的正确与否**智能地切换使用这两种目标**，从而实现对两类样本鲁棒半径 r 的差异化调节，最终增强了模型基于 r 来检测自身错误的能力。

根据论文全文，Radius Aware Training (RAT) 方法的具体训练流程如下：

1.  **核心训练目标**：RAT 的训练目标是提升模型对自身误分类的检测能力，而无需使用任何额外数据（摘要、第4节引言）。

2.  **训练损失函数**：总体训练损失是标准交叉熵损失与RAT特定损失之和（公式(12)）：
    \( L(\Theta; x, y) = L_{CE}(\Theta; x, y) + L_{RAT}(\Theta; x, y) \)
    其中 \( L_{CE} \) 用于维持模型的基础分类准确率，\( L_{RAT} \) 专门用于优化样本的鲁棒半径分布。

3.  **RAT损失（\( L_{RAT} \)）的设计**：这是方法的核心。对于每个训练样本 \( (x, y) \)，模型会先做出预测 \( \hat{y}(x) \)。然后根据该预测是否正确，采用不同的对抗扰动优化目标（公式(11)）：
    *   **对于训练中被模型误分类的样本**（即 \( \hat{y}(x) \neq y \)）：
        应用内层**最小化**目标：\( \min_{\|x' - x\|_{\infty} \leq \epsilon} l(\Theta; x') \)
        其目的是找到一个小扰动，使扰动后样本的损失**变小**，这相当于允许模型对该样本的预测在轻微扰动下变得“正确”，从而**减小**该样本的鲁棒半径，将其“拉近”决策边界。
    *   **对于训练中被模型正确分类的样本**（即 \( \hat{y}(x) = y \)）：
        应用内层**最大化**目标：\( \max_{\|x' - x\|_{\infty} \leq \epsilon} l(\Theta; x') \)
        其目的是找到一个小扰动，使扰动后样本的损失**变大**，这迫使模型对该样本的预测在面对扰动时保持稳定，从而**增大**该样本的鲁棒半径，将其“推离”决策边界。
    *   **直观理解**：如图1（左）所示，此设计旨在通过训练，扩大正确样本与错误样本在鲁棒半径上的差距，使模型在推理时能更易利用半径大小来识别潜在错误。

4.  **优化算法**：论文使用FGSM对抗攻击算法来高效求解上述内层的最大化和最小化问题（第4节）。RAT目标理论上可与任何对抗攻击算法结合，但FGSM因其简单高效被采用。

**总结**：RAT训练是一种将标准分类训练与对抗性训练目标相结合的方法。其关键在于**动态地、有区别地**处理训练样本：对当前预测错误的样本施加“软化”目标以减小其鲁棒半径，对预测正确的样本施加“硬化”目标以增大其鲁棒半径，从而在不依赖额外数据的前提下，系统性地重塑决策边界附近的样本分布，以提升模型后续的误分类检测性能。
