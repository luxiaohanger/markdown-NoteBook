# 总览
> 进程是资源分配的最小单位，线程是cpu调度的最小单位

在Linux系统中，不刻意区分进程和线程，都通过 task_struct 进行调度，进程拥有独立地址空间，线程共享进程的地址空间

## PCB TCB
在操作系统的设计中，**PCB (Process Control Block)** 和 **TCB (Thread Control Block)** 是内核管理任务的“户口本”。

### 1. PCB 与 TCB 的物理位置

PCB 和 TCB 始终存储在 **内核空间（Kernel Space）** 中。

* **安全性**：它们包含敏感的控制信息（如优先级、内存页表指针、CPU 寄存器快照），不能允许用户程序直接修改。
* **常驻内存**：为了保证内核能够随时进行调度，活跃进程的 PCB/TCB 通常常驻在内核内存中，不会被交换到磁盘上。
* **关联关系**：在 Linux 中，PCB 对应 `task_struct` 结构体。如果是多线程进程，每个线程都有自己的 TCB（在 Linux 中同样表现为 `task_struct`），而它们共同指向同一个内存描述符（`mm_struct`）以共享地址空间。


### 2. 进程/线程挂起时：保留 vs 重新加载

当一个进程或线程被**挂起（Suspended/Swapped Out）**，通常意味着它不仅失去了 CPU 执行权，其用户空间的内存数据也可能被置换到磁盘的 **Swap 分区** 中。

#### A. 哪些被保留（通常留在内存中）

为了让内核能够“记得”这个任务并在以后唤醒它，以下信息必须保留在物理内存的内核区：

1. **PCB/TCB 主体**：任务的状态（Suspended）、唯一标识（PID/TID）、退出码等基本信息。
2. **内存映射信息**：页表或指向页表的指针。内核需要知道该进程的数据被换到了磁盘的哪个扇区。
3. **优先级与调度参数**：决定该任务何时能从挂起态转为就绪态。
4. **同步对象控制块**：例如它正在等待的信号量或锁的队列信息。

#### B. 哪些被“丢弃”并需要重新加载

这些数据通常被交换到磁盘，或者在切换时从寄存器中清空，恢复执行时必须重新载入：

1. **CPU 上下文 (Context)**：
* **通用寄存器**、**程序计数器 (PC)**、**状态寄存器**、**栈指针 (SP)**。
* 在挂起瞬间，这些值会被保存到 TCB/PCB 的特定区域；恢复执行时，内核将这些值重新加载回物理 CPU。


2. **用户态栈与数据 (User Memory)**：
* 进程的代码段、数据段、堆空间以及用户栈。
* 挂起时这些可能被移出物理内存。当任务恢复时，如果访问这些内存，会触发 **缺页中断 (Page Fault)**，由内核重新从磁盘加载回物理内存。


3. **TLB 缓存 (针对进程)**：
* 如果是进程切换，由于地址空间变了，硬件 TLB（快表）会被刷新。恢复时需要通过重新查询页表来逐步重建。
