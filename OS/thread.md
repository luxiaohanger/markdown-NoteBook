## Thread的资源
在基于进程的多线程模型中（如Linux），每个线程既有与其他线程共享的资源，也有自己独有的资源。

**线程之间共享的资源（整个进程一份）**：
1.  **内存空间**：
    *   **代码段**：可执行的程序指令。
    *   **数据段**：全局变量、静态变量。
    *   **堆**：动态分配的内存（`malloc` / `new`）。
    *   这些共享内存是线程间通信最直接、最高效的方式，但也**必须通过同步机制**（如互斥锁、信号量）来保护，防止数据竞争。

2.  **进程级资源**：
    *   **文件描述符表**：打开的文件、套接字、管道等。一个线程打开文件，其他线程可直接读写。
    *   **信号处理器**：为各种信号设置的处理函数。
    *   **当前工作目录**。
    *   **用户ID和组ID**。

**每个线程独有的资源**：
1.  **线程ID**：系统内唯一的标识符。
2.  **寄存器状态**：包括程序计数器（PC）、栈指针等，用于保存线程的执行现场。
3.  **栈**：这是最关键的私有资源。每个线程有自己独立的调用栈，用于存储局部变量、函数调用参数和返回地址。这确保了函数调用的独立性。
5.  **信号屏蔽字**：线程可以独立设置阻塞或接收哪些信号。
6.  **错误码`errno`**：在POSIX线程中，`errno`是线程局部的，避免了多线程下错误报告的混乱。

**一个简单的类比**：
把进程比作一个**公司办公室**：
*   **共享的**：办公室的公共空间（内存）、公司银行账户（全局数据）、对外的电话号码（文件描述符）、公司规章制度（代码）。
*   **线程独有的**：每个员工（线程）有自己的办公桌和抽屉（栈）、正在处理的任务进度（寄存器）、员工工牌（线程ID）。

**重要结论**：
正是由于**内存（尤其是堆和全局数据）的共享**，多线程才能高效通信与合作；同时也由于**栈的私有**，保证了函数执行的独立性。编程时，**必须**对共享数据的访问进行同步，而线程的局部变量是天然线程安全的。

## Thread Safety
在多线程编程中，**线程安全（Thread Safety）** 是一个核心概念。简单来说，如果一个函数、库或对象在**多个线程同时访问**时，依然能表现出**正确的行为**，不需要调用者进行额外的协调（如加锁），那它就是线程安全的。

反之，如果多线程同时操作导致了数据错乱、程序崩溃或逻辑错误，我们就称之为**线程不安全**。

---

### 1. 核心矛盾：竞态条件（Race Condition）

线程不安全的根源通常是**竞态条件**。当多个线程同时读写同一个共享变量，且最终结果取决于线程执行的“时机”或“顺序”时，问题就产生了。

---

### 2. 线程安全必须满足的“三大特性”

要实现线程安全，通常需要保证以下三点：

* **原子性（Atomicity）：** 一个操作要么全部执行成功，要么全部不执行，中间过程不能被其他线程中断。
    * *非原子操作：* C 语言中的 `i++` 实际上包含“读取-增加-写入”三个步骤，在多线程下是不安全的。
* **可见性（Visibility）：** 当一个线程修改了共享变量的值，其他线程能够**立即**看到这个修改。
    * *背景：* 正如我们之前讨论的 **TSO/RMM** 模型，CPU 缓存可能导致一个核修改了数据，另一个核还在读旧的缓存。
* **有序性（Ordering）：** 程序执行的顺序按照代码的先后顺序执行，不被编译器或处理器为了优化性能而进行乱序重排。

---

### 3. 如何实现线程安全？

在操作系统和 C 语言底层开发中，常用的手段有：

| 手段 | 描述 |
| :--- | :--- |
| **互斥锁 (Mutex)** | 同一时刻只允许一个线程进入临界区。 |
| **原子操作 (Atomic)** | 使用硬件指令（如 `Compare-And-Swap`）保证操作不可分割。 |
| **线程局部存储 (TLS)** | 每个线程拥有独立的变量副本，不共享自然安全。 |
| **不可变对象 (Immutable)** | 数据一旦创建就不允许修改，只读操作永远是安全的。 |

---

### 4. 常见的陷阱：标准库函数

并不是所有的标准库函数都是线程安全的。例如：
* **`strtok()`**：因为它内部使用了一个静态指针来记录处理进度，多个线程同时调用会互相干扰。
* **`rand()`**：通常内部维护一个全局的随机种子。

> **小贴士：** 在 Linux 下，很多非线程安全的函数都有对应的线程安全版本，通常以 `_r` 结尾（如 `strtok_r`）。
