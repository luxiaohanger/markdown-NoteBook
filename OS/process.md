# process
## pid
每个进程有属于自己的processid，C 中使用（int）getpid() 获得

------
## fork()
fork() 函数调用时创建父进程的副本，子进程的上下文和父进程完全一样，PC也一样，因此子进程会从 fork() 函数的下一行开始执行，而非从main() 函数开始执行。

fork（）函数返回一个整数，调用成功时在父进程中返回子进程的进程PID（失败返回-1），而子进程中由内核设置为0。

## wait()
`wait()` 是一个**系统调用**，其主要作用是**让父进程暂停执行（进入阻塞状态），等待其某个子进程终止，并回收该子进程的资源**。

它的核心功能和目的如下：

1.  **进程同步**：确保父进程在子进程结束之后，再继续执行后续代码。这对于有依赖关系的任务顺序非常重要。
2.  **回收资源**：当一个进程终止时，它并不会立即从系统内完全消失，而是会变成一个“僵尸进程”（Zombie），保留着退出状态等信息，直到其父进程调用 `wait()`（或类似的函数）来读取这些信息。`wait()` 会完成这个“收尸”工作，释放子进程占用的所有系统资源。如果不回收，僵尸进程会长期占用进程号，可能导致系统资源耗尽。
3.  **获取子进程退出状态**：父进程可以通过 `wait()` 的返回值或参数，知道子进程是正常退出（以及退出码），还是被信号终止。

每次调用只等待并回收一个终止的子进程，成功时返回其pid，否则返回 -1

## exec()
`exec()` 是类Unix系统中一组非常重要的系统调用/库函数，用于**执行一个新程序，替换当前进程的镜像**。

简单来说，它的核心功能是：**“换脑不换壳”**。当前进程的“身体”（进程号、环境、打开的文件描述符等）还在，但“脑子”（正在执行的程序代码和数据）被完全替换成了一个新的程序。

### 核心特性
1.  **进程替换，而非创建**：它不会创建一个新进程，而是在**当前进程**的上下文中加载并运行一个全新的程序。新程序从`main()`函数开始执行，进程ID（PID）保持不变。
2.  **成功不返回**：如果`exec()`调用成功，**它将永远不会返回到调用它的原程序**。原程序剩余的代码都被新程序覆盖了。
3.  **失败才返回**：只有调用失败时（如找不到指定的可执行文件），它才会返回一个错误值（通常是-1），并设置相应的`errno`。

### 常见函数变体
`exec()`函数族包含多个以不同字母后缀结尾的函数，它们主要在**参数传递方式**上有所不同：

| 函数原型 | 参数列表方式 | 环境变量 |
| :--- | :--- | :--- |
| `int execl(const char *path, const char *arg, ..., NULL)` | **列表** | 继承当前环境 |
| `int execv(const char *path, char *const argv[])` | **数组** | 继承当前环境 |
| `int execle(const char *path, const char *arg, ..., NULL, char *const envp[])` | 列表 | **指定新环境** |
| `int execve(const char *path, char *const argv[], char *const envp[])` | 数组 | 指定新环境 |
| `int execlp(const char *file, const char *arg, ..., NULL)` | 列表 | 继承环境，在PATH中查找文件 |
| `int execvp(const char *file, char *const argv[])` | 数组 | 继承环境，在PATH中查找文件 |

**字母含义**：
*   **`l` (list)**：参数以可变参数列表（`arg0, arg1, ..., NULL`）形式传递，`NULL`结束。
*   **`v` (vector)**：参数以字符串数组 `argv[]` 形式传递，数组最后一个元素必须是`NULL`。
*   **`p` (PATH)**：第一个参数是**文件名**，系统会在`PATH`环境变量指定的目录中搜索这个可执行文件。不带`p`的版本，第一个参数必须是**完整路径**。
*   **`e` (environment)**：可以传递一个全新的环境变量数组 `envp[]` 给新程序。不带的则继承调用进程的所有环境变量。

**`execve` 是真正的系统调用**，其他函数都是基于它封装的库函数。

### 基本用法示例
下面是一个在C语言中使用`execl`和`execvp`的简单例子：

```c
#include <unistd.h>
#include <stdio.h>
#include <errno.h>

int main() {
    printf("原进程（PID: %d）即将启动 ls 命令。\n", getpid());

    // 示例1: 使用 execl
    // 参数以列表形式给出，必须提供ls的完整路径
    execl("/bin/ls", "ls", "-l", "-a", NULL);

    // 示例2: 使用 execvp (更常用)
    // 参数以数组形式给出，只需提供文件名，系统会去PATH里找
    // char *args[] = {"ls", "-l", "-h", NULL};
    // execvp("ls", args);

    // 如果 exec 成功，下面的代码永远不会执行
    // 只有失败时才会运行到这里
    perror("exec 失败");
    return 1;
}
```

### 经典组合：fork() + exec()
单独使用`exec()`会替换掉当前进程，这通常不是我们想要的。更常见的模式是先用 `fork()` 创建一个子进程，然后在**子进程**中调用`exec()`来运行新程序，父进程则继续执行原有逻辑。这是Shell运行命令、服务器创建守护进程等场景的基础。

```c
#include <unistd.h>
#include <sys/wait.h>
#include <stdio.h>

int main() {
    pid_t pid = fork();

    if (pid == 0) {
        // 子进程
        printf("子进程中...\n");
        char *args[] = {"sleep", "5", NULL};
        execvp("sleep", args);
        // 如果 exec 失败
        perror("execvp");
        _exit(1); // 子进程异常退出
    } else if (pid > 0) {
        // 父进程
        printf("父进程（PID: %d）等待子进程结束。\n", getpid());
        wait(NULL); // 等待子进程
        printf("子进程执行完毕。\n");
    } else {
        perror("fork");
        return 1;
    }
    return 0;
}
```

### 注意事项
1.  **参数惯例**：`argv[0]` 传统上是程序名本身，虽然新程序可以忽略或使用它。
2.  **文件描述符**：默认情况下，调用`exec()`后，原进程打开的文件描述符（除了标记了`FD_CLOEXEC`的）仍然保持打开状态。这对于实现输入/输出重定向至关重要。
3.  **信号处理**：被捕获的信号会重置为默认行为；被忽略的信号则保持忽略。
