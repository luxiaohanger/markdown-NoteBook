# Memory Model
在操作系统和处理器架构中，内存模型（Memory Model）定义了多核系统中读写操作的可见性顺序。理解 **TSO（Total Store Ordering）** 和 **RMM（Relaxed Memory Model）** 的关键在于理解硬件为了性能究竟“偷跑”了多远。

---

## 1. TSO vs. RMM：核心区别

### **TSO (Total Store Ordering - 全存储排序)**
TSO 是一种**较强**的内存模型，最为人熟知的实现是 **x86** 架构。
* **核心逻辑：** 它允许“写-读”重排序。也就是说，一个处理器可以先看到自己刚写进缓冲区的值，而不必等待该值同步到主内存。
* **限制：** 所有的写操作（Stores）必须按照它们在程序中出现的顺序进入全局队列。
* **直观理解：** 就像所有 CPU 共用一个先进先出的写缓冲区。你发出的指令顺序是 A、B、C，那么所有人看到你修改生效的顺序也一定是 A、B、C。

### **RMM (Relaxed Memory Model - 松散内存模型)**
RMM（常见于 **ARM** 或 **RISC-V**）是一种**弱**内存模型，为了榨取硬件性能，它赋予了硬件极大的重排序自由。
* **核心逻辑：** 只要不违反单线程内的逻辑依赖，硬件可以随意重排序“读-读”、“读-写”、“写-读”和“写-写”。
* **开发者负担：** 硬件不再保证顺序，开发者必须显式地使用 **Memory Barrier（内存屏障，如 `dmb`, `lwsync`）** 来强制规定某些操作的先后顺序。
* **直观理解：** 每个核都有自己的节奏。除非你大喊一声“停！前面的没写完不许往后走”，否则后发的指令完全可能先被其他核观察到。

| 特性 | TSO (e.g., x86) | RMM (e.g., ARM/RISC-V) |
| :--- | :--- | :--- |
| **重排序自由度** | 低（仅允许 Store-Load 重排） | 高（几乎所有组合皆可重排） |
| **性能潜力** | 中 | 高（硬件调度更灵活） |
| **编程难度** | 较低 | 较高（需频繁处理屏障指令） |

## 2. 什么是“一致”与“全局”

这两个词通常出现在讨论 **Cache Coherence（缓存一致性）** 和 **Memory Consistency（内存一致性）** 的上下文中。

### **“一致” (Coherence)**
“一致”通常指**缓存一致性（Cache Coherence）**。它关注的是**单一内存地址**在多个 CPU 缓存中的同步。
* **目标：** 确保所有的处理器对同一个变量看到的是同一个值。
* **底层机制：** 比如经典的 MESI 协议。当 CPU A 修改了地址 `0x100`，它必须通知 CPU B 使其缓存中的旧值失效。
* **总结：** 一致性保证了“你读到的不是旧数据”。

### **“全局” (Global Observability / Global Order)**
“全局”通常指**全局可见性顺序**。它关注的是**多个不同内存地址**的操作在所有处理器面前呈现出的顺序。
* **概念：** 当一个写操作被标记为“全局可见”时，意味着系统中所有的核心都已经观察到了这次修改。
* **在 TSO 中：** 存在一个单一的全局顺序，所有核看到的写操作序列是一模一样的。
* **在 RMM 中：** 可能不存在统一的全局顺序。CPU A 看到的变更顺序可能是 `X -> Y`，而 CPU B 观察到的却是 `Y -> X`（如果它们之间没有同步的话）。
