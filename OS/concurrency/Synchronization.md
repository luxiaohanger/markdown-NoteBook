# 同步
在多线程编程（Concurrency）中，Wait（等待） 和 Wakeup（唤醒，通常指 notify 或 signal） 是实现线程间通信（Inter-Thread Communication）的核心机制。

简单来说，它们让线程能够“有礼貌”地协调工作：当一个线程需要的条件还没满足时，它就先睡一会（Wait）；当另一个线程创造了条件后，再把它叫醒（Wakeup）。

-------

## 丢失唤醒 (Lost Wake-up Problem)

丢失唤醒发生在 **“检查条件”** 和 **“进入睡眠”** 这两个动作之间，由于缺乏原子性，导致唤醒信号在线程进入睡眠之前就已经发出，从而使线程永久阻塞。

### 发生场景
假设有一个生产者和消费者共享的变量 `count`。
1.  **消费者：** 检查 `count == 0`，准备去睡觉。
2.  **上下文切换：** 此时 CPU 切换到了生产者。
3.  **生产者：** 生产了一个东西，设置 `count = 1`，并发出了唤醒信号（Signal）。但此时消费者还没睡着，这个信号被丢弃了。
4.  **上下文切换：** 消费者回到 CPU，它认为刚才检查的结果是 0，于是执行 `sleep()`。
5.  **死锁：** 消费者在等待一个永远不会再来的信号，而生产者可能也在等待消费者消费完后再生产，系统卡死。
