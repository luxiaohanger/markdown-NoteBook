# Dynamic Programming
## 特点
1.`最优子结构`

2.`无后效性`
已经求解的子问题，不会再受到后续决策的影响。

3.`子问题重叠`
如果有大量的重叠子问题，我们可以用空间将这些子问题的解存储下来，避免重复求解相同的子问题，从而提升效率。

## 基本思路
1.将原问题划分为若干 `阶段`，每个阶段对应若干个子问题，提取这些子问题的特征（称之为 `状态`）；

2.寻找每一个状态的可能 `决策`，或者说是各状态间的相互转移方式（用数学的语言描述就是 `状态转移方程`）。

3.按顺序求解每一个阶段的问题。

```
确定dp数组（dp table）以及下标的含义
确定递推公式
dp数组如何初始化
确定遍历顺序与边界条件
举例推导dp数组
```

## 小巧思
1.要么猜状态转移方程，要么猜 dp 数组，这个建模方式是最重要的,注意分析解的结构来分析

2.看大问题的最后一步是什么，可能是最优子结构的合并，由此来找最优子结构

3.状态转移不一定是一个状态直接得到另一个，可能是多个状态的和，可能是遍历所有可能子状态的极值

4.常见的状态：某个元素是否在解中（可能是直接作为DP下标，也可能是状态转移的讨论分支）；某个范围（0 - i）是否在解中；

5.分维度构建DP，看看要表达的状态有多少，就有几个维度

## 两种方式
`自顶向下`

从最终的大问题（m[1, n]）开始，递归地分解成小问题。如果遇到重复的子问题，就查表避免重复计算。

`自底向上`

从最小的子问题开始，主动地、按顺序把所有可能的子问题都算一遍，最后自然就得到了大问题的解。

此时填表的顺序就很重要了


## 常见模型
### 01背包
常见形式：选取容器内其中一些元素，每个元素只能被挑选一次，每个元素占有一定体积，总体积固定不得超出，每个元素带有一些价值，求满足条件的情况（最大值、恰好是）

标准模板：dp[i][j] 容量为 i 的背包，候选元素为 0 - j，最大价值
```cpp
for(int i = 1;i<=n;++i) {
    if(i >= sp[0])dp[i][0] = val[0];
}

for(int i = 1;i<=n;++i) {
        for(int j = 1;j<m;++j) {
            //select j
            if(i - sp[j] >= 0)dp[i][j] = max(dp[i][j],dp[i - sp[j]][j - 1] + val[j]);
            //dont select j
            dp[i][j] = max(dp[i][j],dp[i][j - 1]);
        }
}
```

一维压缩：dp[i] 容量为i的背包，能装的最大价值
```cpp
for(int j = 0;j < m;++j){
    //依次解锁物品 j
    for(int i = n;i >= sp[j];--i){
        //倒序遍历，确保之前的没用过
        dp[i] = max(dp[i],dp[i - sp[j]] + val[j]);
    }
}
```

变式：

- 能否选出某条件的元素集（给定背包容积，价值 = 容积，判断最大价值是否等于容积）
- 选择满足条件元素子集的方法数（在数组中找和为s的子集数）：
dp[i]定义为装满容积为i背包的方法数
```cpp
    dp[0] = 1;
    for(auto x:nums) {
        //一维dp数组，外层遍历数组，表示解锁这个数
        for(int i = s;i >= x;--i){
            //状态转移，从没用过 x 并且容量为 i-x 转移到 i
            //倒序遍历，确保之前的没用过 x
            dp[i] += dp[i - x];
        }
    }
    return dp[s];
```
