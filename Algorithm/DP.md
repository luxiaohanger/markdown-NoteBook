# Dynamic Programming
## 特点
1.`最优子结构`

2.`无后效性`
已经求解的子问题，不会再受到后续决策的影响。

3.`子问题重叠`
如果有大量的重叠子问题，我们可以用空间将这些子问题的解存储下来，避免重复求解相同的子问题，从而提升效率。

## 基本思路
1.将原问题划分为若干 `阶段`，每个阶段对应若干个子问题，提取这些子问题的特征（称之为 `状态`）；

2.寻找每一个状态的可能 `决策`，或者说是各状态间的相互转移方式（用数学的语言描述就是 `状态转移方程`）。

3.按顺序求解每一个阶段的问题。

```
确定dp数组（dp table）以及下标的含义
确定递推公式
dp数组如何初始化
确定遍历顺序与边界条件
举例推导dp数组
```

## 小巧思
1.要么猜状态转移方程，要么猜 dp 数组，这个建模方式是最重要的,注意分析解的结构来分析

2.看大问题的最后一步是什么，可能是最优子结构的合并，由此来找最优子结构

3.状态转移不一定是一个状态直接得到另一个，可能是多个状态的和，可能是遍历所有可能子状态的极值

4.可以从找状态转移入手，也即找一个动作，一般是选取了某个元素，导致某个状态改变

5.分维度构建DP，看看要表达的状态有多少，就有几个维度

## 两种方式
`自顶向下`

从最终的大问题（m[1, n]）开始，递归地分解成小问题。如果遇到重复的子问题，就查表避免重复计算。

对于树形结构等不易于自底向上填表的情况，可以用哈希表存储子问题的解，进行记忆化递归，为了保证每个节点最多访问一次，可以使用如下写法：

```cpp
unordered_map<TreeNode*,int> umap;

int func(TreeNode* root){
    if(!root)return default_val;
    if(umap[root])return umap[root];
    //process
    umap[root] = val;
    return umap[root];
}
```

`自底向上`

从最小的子问题开始，主动地、按顺序把所有可能的子问题都算一遍，最后自然就得到了大问题的解。通常要求大问题由子问题转移而来。

此时填表的顺序就很重要了

## 常见dp定义及其转移
1.dp[i][j]表示下标范围 i - j 的元素满足条件，`dp[i][j] && dp[j + 1][k] --> dp[i][k]`






## 常见模型
背包模型的本质是在一个元素列表中挑选一些元素，根据元素的可重复性分为不同背包，元素有两个属性，代价和价值，我们的背包就是一定代价容量的背包，我们要根据要求选取一定代价范围内的元素，而这类挑选问题的状态转移的本质就是：`某个代价价值状态 + 选择某个元素 --> 新的代价价值状态`
### 01背包
常见形式：选取容器内其中一些元素，每个元素只能被挑选一次，每个元素占有一定体积，总体积固定不得超出，每个元素带有一些价值，求满足条件的情况（最大值、恰好是）

标准模板：dp[i][j] 容量为 i 的背包，候选元素为 0 - j，最大价值
```cpp
for(int i = 1;i<=n;++i) {
    if(i >= sp[0])dp[i][0] = val[0];
}

for(int i = 1;i<=n;++i) {
        for(int j = 1;j<m;++j) {
            //select j
            if(i - sp[j] >= 0)dp[i][j] = max(dp[i][j],dp[i - sp[j]][j - 1] + val[j]);
            //dont select j
            dp[i][j] = max(dp[i][j],dp[i][j - 1]);
        }
}
```

一维压缩：dp[i] 容量为i的背包，能装的最大价值
```cpp
for(int j = 0;j < m;++j){
    //依次解锁物品 j
    for(int i = n;i >= sp[j];--i){
        //倒序遍历，确保之前的没用过
        dp[i] = max(dp[i],dp[i - sp[j]] + val[j]);
    }
}
```

变式：

- 能否选出某条件的元素集（给定背包容积，价值 = 容积，判断最大价值是否等于容积）
- 选择满足条件元素子集的方法数（在数组中找和为s的子集数）：
dp[i]定义为装满容积为i背包的方法数
```cpp
    dp[0] = 1;
    for(auto x:nums) {
        //一维dp数组，外层遍历数组，表示解锁这个数
        for(int i = s;i >= x;--i){
            //状态转移，从没用过 x 并且容量为 i-x 转移到 i
            //倒序遍历，确保之前的没用过 x
            dp[i] += dp[i - x];
        }
    }
    return dp[s];
```

### 完全背包
每个物品选择次数不限，给定背包容量，求最大价值

标准一维版本：dp[i]容量为 i 的背包最大价值，做法和01背包空间压缩思路相似，不过内层遍历背包容量时正序遍历，表示可以重复选择
```cpp
for(int i = 0;i<n;++i) {
    for(int j = sp[i];j <= v;++j) {
        //在遍历中限制范围，减少判断
        dp[j] = max(dp[j],dp[j - sp[i]] + val[i]);
    }
}
```

变式：

- 满足条件的选取方法数
方法数问题就是把状态转移改为 `dp[i] += dp[i - x]`,但这里需要注意两层循环的顺序问题，如果结果是组合数，也即对于元素间顺序没有要求，则外层是遍历元素，内层遍历容量，此时每次解锁一个元素，考察对于选取这个元素的所有方法数，不涉及选取元素的顺序问题；如果结果是排列数，也即对于元素间的顺序有要求，则反之，此时内层依次遍历每个元素，对于同一个容量 x + y ,既包含了 {x,y} ,也包含了 {y,x}

推广：只要选取的元素集对于排列顺序有要求，例如字符串拼接，把元素列举放在内部

### 多重背包
每个物品限制使用次数，如果把每次分开，就变成了 01背包


### 分组背包
将物品分为若干组，每组最多选择一个
```cpp
// 输入：背包总容量 V， 组数 K，以及每组物品的信息（体积v，价值w）
// 初始化：dp数组全为0
vector<int> dp(V + 1, 0);

for (int i = 1; i <= K; ++i) { // 遍历每一组
    for (int j = V; j >= 0; --j) { // 背包容量逆序遍历,低容量背包后更新，保证每组最多一个（01背包思想）
        for (int k = 0; k < 第i组的物品数量; ++k) { // 遍历组内每个物品
            if (j >= v[i][k]) { // 如果当前背包容量能放下这个物品
                dp[j] = max(dp[j], dp[j - v[i][k]] + w[i][k]);
            }
        }
    }
}
// 最终答案：dp[V]
```
### 间隔取数问题
给定有序元素列表，相邻元素不能同时取，求最优解

一种思路是不陷入某一个到底取不取的讨论，让dp自动去判断
```cpp
for(int i = 2;i<n;++i){
    dp[i] = max(dp[i - 1],dp[i - 2] + nums[i]);
    //在状态转移中限定间隔的性质即可
}
```

另一种做法是在dp数组中记录元素是否取，一般需要双倍空间
```cpp
//dp[i][0]no  dp[i][1] yes
for(int i = 1;i<n;++i){
    dp[i][0] = max(dp[i - 1][0],dp[i - 1][1]);
    dp[i][1] = dp[i - 1][0] + nums[i];
}

return max(dp[n - 1][0],dp[n - 1][1]);
```


### 股票问题
prices[i] 表示第 i 天股票的价格，求在给定限制条件下的最大收益

dp[i][k] 表示第 i 天的状态（持有，非持有，卖出，冷冻，第几次操作）
- 限制购买次数为 k 次，并且手头最多同时存在一支股票
dp[i][2 * k - 1] 表示第 i 天持有第 k 支股票的最大现金剩余（前一天也持有/前一天不持有第 k - 1 支今日买入），dp[i][2 * k]表示第 i 天不持有第 k 支股票的最大现金剩余（前一天不持有/前一天持有第 k 支今日卖出）


```cpp
for (int i = 1; i < 2 * k; i += 2) {
        dp[0][i] = -prices[0];
    }

    for (int i = 1; i < prices.size(); ++i) {
        for (int j = 1; j < 2 * k; j += 2) {
            dp[i][j] = max(dp[i - 1][j], dp[i - 1][j - 1] - prices[i]);
            dp[i][j + 1] = max(dp[i - 1][j + 1], dp[i - 1][j] + prices[i]);
        }
    }
```
- 卖出后限制冷冻天数
每天的状态分为：持有、卖出、非卖出的非持有、冷冻，把非持有分成有无卖出是为了便于冷冻的转移
