# 递归

### Master定理
对于形如 \( T(n) = aT(n/b) + f(n) \) 的递归方程，其中：
- \( a \geq 1 \) 表示子问题的数量，
- \( b > 1 \) 表示问题规模的缩小因子，
- \( f(n) \) 表示除了递归调用外的额外工作量（如分解和合并的开销）。

定理通过比较 \( f(n) \) 与 \( n^{\log_b a} \) 的渐近行为，将解分为三种情况：
- 如果 \( f(n) = O(n^{\log_b a - \epsilon}) \) 对于某个常数 \( \epsilon > 0 \)，则 \( T(n) = \Theta(n^{\log_b a}) \)


- 如果 \( f(n) = \Theta(n^{\log_b a} )\) ，则 \( T(n) = \Theta(n^{\log_b a} \log n) \)


- 如果 \( f(n) = \Omega(n^{\log_b a + \epsilon}) \) 且满足正则条件 \( af(n/b) \leq cf(n) \) 对于某个常数 \( c < 1 \) 和所有足够大的 \( n \)，则 \( T(n) = \Theta(f(n)) \)



### 不符合Master定理的递归式的求解方法
当递归式不满足Master定理的条件时（例如，形式不匹配或 \( f(n) \) 与 \( n^{\log_b a} \) 无法直接比较），可以采用以下方法：
#### 1.递归树法
通过展开递归式，构建树状结构来直观计算每一层的工作量，然后求和得到总复杂度。例如，对于 \( T(n) = T(n/3) + T(2n/3) + n \)，可以绘制递归树并分析各层成本，最终推导出 \( T(n) = O(n \log n) \)。

首先求每层的工作量，也即每层的多个 \(T(n/b_i)\) 所对应的 \( \sum f(n/b_i)\)

再求递归树的树高，由最长的路径决定，观察哪个递归缩减的最慢，用最慢的分支来求树高

最后对级数求和

#### 2.变量代换法（将不熟悉的形式转化为熟悉的形式）

适用于递归式中出现`n`的指数、对数等非标准形式。

构造同构形式

**举例：`T(n) = 2T(√n) + log n`**

**解题思路：**
1.  **代换以消除根号**：令 `m = log n`，则 `n = 2^m`。同时，`√n = n^(1/2) = (2^m)^(1/2) = 2^(m/2)`。
2.  **改写递归式**：
    - 原式`T(n) = 2T(√n) + log n` 变为 `T(2^m) = 2T(2^(m/2)) + m`。
    - 令 `S(m) = T(2^m)`，则得到新递归式：**`S(m) = 2S(m/2) + m`**。
3.  **求解新递归式**：新形式`S(m)=2S(m/2)+m`**完美符合Master定理**！
    - 这里 `a=2, b=2, f(m)=m`。
    - `n^(log_b a) = m^(log_2 2) = m^1 = m`。
    - 属于**情况2**：`f(m) = Θ(m)`，所以 `S(m) = Θ(m log m)`。
4.  **代回原变量**：
    - 因为 `S(m) = Θ(m log m)`，且 `m = log n`，所以 `T(n) = S(log n) = Θ(log n * log log n)`。

**主导项分析**：通过代换，我们将一个关于`n`的复杂递归，转换成了一个关于`log n`的、结构清晰的递归。主导项从`f(n)=log n`和递归结构共同作用，产生了`log n * log log n`。

#### 3.代入法/数学归纳法（当你能“猜”出解时，用于严格证明）

当你凭经验或直觉猜测出一个渐进复杂度时，可以用代入法严格证明。

**举例：`T(n) = 2T(n-1) + 1`，且`T(1)=1`**（这是典型的汉诺塔递归式）

**解题思路：**
1.  **观察与猜测**：
    - `T(1) = 1`
    - `T(2) = 2*1 + 1 = 3`
    - `T(3) = 2*3 + 1 = 7`
    - `T(4) = 2*7 + 1 = 15`
    - 规律：`T(n) = 2^n - 1`。所以我们**猜测** `T(n) = O(2^n)`，更精确地说是 `T(n) = 2^n - 1`。
2.  **归纳证明**：
    - **归纳基础**：`n=1`时，`T(1)=1=2^1-1`，成立。
    - **归纳假设**：假设对于所有 `k < n`，`T(k) = 2^k - 1` 成立。
    - **归纳步骤**：
            T(n) = 2T(n-1) + 1
                 = 2 * (2^(n-1) - 1) + 1  // 应用归纳假设于T(n-1)
                 = 2^n - 2 + 1
                 = 2^n - 1
    - 因此，`T(n) = 2^n - 1`对所有`n`成立，精确解为`Θ(2^n)`。

**主导项分析**：这是一个线性齐次递推关系，主导项由特征方程决定。递归部分`2T(n-1)`直接导致了指数级增长`2^n`，常数项`+1`在渐进意义上被吸收。



## 算法正确性证明
- **关键不变量的设计与表述**：在算法设计中，不变量（如循环不变式）是用于描述算法执行过程中始终保持为真的性质。关键点包括：
  - **变量语义**：明确变量在算法中的含义，例如，它们代表已处理的数据、未处理的集合等。
  - **已排除集合与仍可能集合**：在不变量中，常区分已处理部分（如已排序元素）和待处理部分（如未排序元素），这有助于跟踪算法进度。
- **形式化正确性论证的结构**：证明算法正确性通常采用结构化论证，即“初始化-保持-终止”的逻辑闭环。这确保从算法开始到结束，不变量始终成立，从而推导出正确结果。

#### 循环不变式
- **循环不变式的概念与定义**：
  - 循环不变式是算法循环中每次执行前后都保持为真的条件或性质。它描述了循环的中间状态，帮助连接初始条件和最终目标。
  - 定义方法：通常基于算法目标，用数学或逻辑表达式表述。例如，在排序算法中，循环不变式可能是“每次迭代后，前k个元素已按升序排列”。
- **用循环不变式证明算法正确性的三段论**：
  - **初始化**：`证明在循环第一次迭代开始前，循环不变式成立。`这相当于基础步骤，确保算法起点满足不变量。
  - **保持**：`假设在某次迭代开始时循环不变式成立，证明执行循环体后，不变式仍然成立。`这体现了循环的推进不会破坏不变性，是归纳步骤。
  - **终止**：`当循环结束时，结合循环不变式和终止条件，推导出算法结果正确。`例如，在排序算法中，终止时可能所有元素已处理，不变量导出整个序列有序。
