# Dynamic Programming
## 特点
1.`最优子结构`

2.`无后效性`
已经求解的子问题，不会再受到后续决策的影响。

3.`子问题重叠`
如果有大量的重叠子问题，我们可以用空间将这些子问题的解存储下来，避免重复求解相同的子问题，从而提升效率。

## 基本思路
1.将原问题划分为若干 `阶段`，每个阶段对应若干个子问题，提取这些子问题的特征（称之为 `状态`）；

2.寻找每一个状态的可能 `决策`，或者说是各状态间的相互转移方式（用数学的语言描述就是 `状态转移方程`）。

3.按顺序求解每一个阶段的问题。

```
确定dp数组（dp table）以及下标的含义
确定递推公式
dp数组如何初始化
确定遍历顺序与边界条件
举例推导dp数组
```

## 技巧
1.要么猜状态转移方程，要么猜 dp 数组，这个建模方式是最重要的,注意分析解的结构来分析

2.看大问题的最后一步是什么，可能是最优子结构的合并，由此来找最优子结构

3.状态转移不一定是一个状态直接得到另一个，可能是多个状态的和，可能是遍历所有可能子状态的极值

4.可以从找状态转移入手，也即找一个动作，一般是选取了某个元素，导致某个状态改变

5.分维度构建DP，看看要表达的状态有多少，就有几个维度

6.某些状态转移是找递归形式的定义，例如某个性质经过什么操作以后仍然是这个性质，那么就形成了状态转移，并且dp数组一般是bool数组

## 两种方式
`自顶向下`

从最终的大问题（m[1, n]）开始，递归地分解成小问题。如果遇到重复的子问题，就查表避免重复计算。

对于树形结构等不易于自底向上填表的情况，可以用哈希表存储子问题的解，进行记忆化递归，为了保证每个节点最多访问一次，可以使用如下写法：

```cpp
unordered_map<TreeNode*,int> umap;

int func(TreeNode* root){
    if(!root)return default_val;
    if(umap[root])return umap[root];
    //process
    umap[root] = val;
    return umap[root];
}
```

`自底向上`

从最小的子问题开始，主动地、按顺序把所有可能的子问题都算一遍，最后自然就得到了大问题的解。通常要求大问题由子问题转移而来。

此时填表的顺序就很重要了

## 常见dp定义及其转移
1.dp[i][j]表示下标范围 i - j 的元素满足条件，`dp[i][j] && dp[j + 1][k] --> dp[i][k]`

## 易错点
1.dp数组初始化的选择，如果是max转移，就初始化为小值，反之为大值

2.不要死扣线性复杂度，可能最优复杂度是非线形的

3.dp 定义为从 0 - i 范围内的解时，要注意需不需要以 i 结尾,不以结尾的初始化注意某个位置成立以后，后面都要设为成立,或者继承前面的值

4.自底向上构建答案时注意遍历方向，作为转移前状态的一定要先访问

5.注意ans初始化时考虑极端情况，例如无法进入处理过程时是否成立，例如有的答案最小为1而非0
