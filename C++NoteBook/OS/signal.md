在 C++ 开发中，**信号 (Signal)** 是操作系统在发生特定事件时发送给程序的一个“软中断”。


---

## 1. 什么是信号？

信号是由操作系统或硬件产生的异步通知。常见的信号包括：

| 信号名 | 触发条件 | 默认动作 |
| :--- | :--- | :--- |
| **`SIGINT`** | 用户按下 `Ctrl+C` | 终止进程 |
| **`SIGTERM`** | 操作系统发送请求（杀掉进程） | 终止进程 |
| **`SIGFPE`** | 算术错误（如**除以零**） | 终止进程并生成核心转储 |
| **`SIGSEGV`** | 段错误（如访问非法内存、空指针） | 终止进程 |
| **`SIGABRT`** | 程序异常终止（通常由 `abort()` 调用） | 终止进程 |

---

## 2. 核心语法：`signal` 函数

在 C++ 中，我们主要使用 `<csignal>`（C语言中为 `<signal.h>`）头文件来管理信号。

### 函数原型
```cpp
void (*signal(int sig, void (*handler)(int)))(int);
```
这看起来很复杂，拆解开来就是：
1.  **`sig`**: 你要监听的信号编号（如 `SIGINT`）。
2.  **`handler`**: 你定义的**处理函数**。当信号发生时，程序会自动跳到这个函数执行。

---

## 3. 实战示例：捕获 Ctrl+C

让我们写一个程序，让它在用户按下 `Ctrl+C` 时不退出，而是打印一句“抓到你啦！”。

```cpp
#include <iostream>
#include <csignal>   // 必须包含这个头文件
#include <unistd.h>  // 为了使用 sleep 函数

// 1. 定义信号处理函数
// 函数必须接收一个整型参数（信号编号）
void signalHandler(int signum) {
    std::cout << "\n[信号中断] 捕获到信号: " << signum << std::endl;
    
    // 如果不在这里调用 exit()，程序会继续运行
    std::cout << "清理资源中... 程序即将退出。" << std::endl;

    //exit是在进程级别请OS回收资源，会销毁调用他的主函数
    exit(signum);  
}

int main() {
    // 2. 注册信号监听
    // 告诉系统：如果收到 SIGINT，请去执行 signalHandler
    signal(SIGINT, signalHandler);  

    std::cout << "程序正在运行... 按下 Ctrl+C 试试。" << std::endl;

    while(true) {
        std::cout << "处理任务中..." << std::endl;
        sleep(1); 
    }

    return 0;
}
```

## 4. 进阶用法：`raise` 函数

除了被动接收系统的信号，你的程序也可以**主动**给自己发信号。这通常用于模拟错误测试。

```cpp
#include <csignal>
#include <iostream>

int main() {
    std::cout << "准备触发自杀信号..." << std::endl;
    
    // 给自己发送一个终止信号
    raise(SIGTERM); 
    
    std::cout << "这行代码永远不会被执行。" << std::endl;
    return 0;
}
```

## 5. 注意事项（初学者必看）

信号处理虽然强大，但它在工程中有很多“坑”：

1.  **不可重入性**：在信号处理函数内部，**不要调用复杂的函数**（如 `std::cout` 或 `malloc`）。
    * *原因*：如果在执行 `std::cout` 时突然来了信号，而信号处理函数里又调了 `std::cout`，会导致死锁或崩溃。
    * *专业做法*：在处理函数里只修改一个全局标志变量（类型通常为 `volatile sig_atomic_t`），然后在主循环里判断这个变量。

2.  **`SIGKILL` 不能被捕获**：
    * 信号 `9` (`SIGKILL`) 是强制杀掉进程。程序无法拦截它，就像“斩立决”无法求情一样。

3.  **信号处理是全局的**：
    * 一旦设置，整个进程（包括所有线程）都会受影响。

## 总结笔记

* **头文件**：使用 `#include <csignal>`。
* **注册**：使用 `signal(信号名, 处理函数)`。
* **触发**：外部（系统/用户）或内部（使用 `raise`）。
* **用途**：最常用于**优雅退出**（释放内存、关闭数据库连接、保存日志）。
