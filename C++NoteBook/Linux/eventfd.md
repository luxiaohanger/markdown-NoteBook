# 📝 工业级内核笔记：`eventfd` 系统调用深度解析

## 一、 核心 API 规范

### 1. 创建函数

```cpp
#include <sys/eventfd.h>
int eventfd(unsigned int initval, int flags);

```

* **`initval`**：内核计数器（`uint64_t`）的初始值。在我们的 `EventLoop` 唤醒场景中，一律初始化为 `0`。
* **`flags`（工业级标配组合）**：
* `EFD_NONBLOCK`：设置为非阻塞 I/O。防止在计数器满（write 阻塞）或计数器为空（read 阻塞）时把你的主线程卡死。
* `EFD_CLOEXEC`：执行 `exec` 系统调用时自动关闭该 fd，防止文件描述符泄漏给子进程。
* *(注：千万不要加 `EFD_SEMAPHORE`，否则 read 就会变成每次只减 1，直接导致你的 ET 模式死锁)*。



### 2. 写入操作（`write`）

* **规范**：传入的缓冲区必须严格等于 **8 字节**（`sizeof(uint64_t)`）。
* **内核行为**：数值累加（加法器）。

$$\text{Counter} = \text{Counter} + X$$


* **返回值 `n**`：
* `n == 8`：成功写入，内核计数器已递增。如果计数器从 0 变为大于 0，且注册了 Epoll ET，将**触发一次**可读信号。
* `n == -1`：失败。若 `errno == EAGAIN`，说明计数器溢出（非阻塞模式特有）；若 `errno == EINVAL`，说明你传入的字节数不是 8。



### 3. 读取操作（`read`）

* **规范**：传入的缓冲区必须严格等于 **8 字节**。
* **内核行为**：**一次性清零**（拔掉阀门）。将当前计数器的累加总和拷贝给用户的变量，随后内核直接将内部计数器**重置为 0**。
* **返回值 `n**`：
* `n == 8`：成功读取。内核计数器已归零，此时 fd 状态变为空，**ET 状态成功复位**。
* `n == -1`：失败。若 `errno == EAGAIN`，说明当前计数器本就是 0，无数据可读；若 `errno == EINTR`，说明被系统信号中断，需要重试。
