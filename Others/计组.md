# 计组
## 运算方法和部件

1.补码整数是模运算系统，因此对运算结果取模，这是高位进位忽略的本质。它所能表示的所有整数，都在这模运算环上，即 [-2ⁿ⁻¹, 2ⁿ⁻¹ - 1]

2.变形补码右侧符号位用于检测溢出，当`两个符号位不一致时发生溢出`。加法运算中，如果两个正数相加结果为正，但符号位变为01，则表示正溢出（结果太大）；如果两个负数相加结果为负，但符号位变为10，则表示负溢出（结果太小）

3.`移码将真实的指数值映射为一个无符号整数`。在移码表示下，指数值的真实大小关系，与它们的移码二进制码的无符号大小关系是完全一致的。因此，CPU的浮点运算单元可以直接使用无符号整数比较器来快速判断两个指数的大小，从而立即确定对阶时尾数的移位方向，极大地提高了运算效率。

4.C语言赋值：位数相同时机器码一致，位数扩展时高位按源的符号扩展（等号右侧）或零扩展，位数减少时直接截断高位


同一表达式不同类型：
`整数提升`：首先，所有小于int的类型（如char、short）被提升为int或unsigned int。
`平衡转换`：之后，如果两个操作数类型仍不同，则按以下规则确定共同类型：
如果两个操作数都是有符号或无符号，但精度不同，则低精度类型转换为高精度类型。
如果一个有符号、一个无符号，且无符号类型的精度大于或等于有符号类型的精度，则有符号数转换为无符号类型。

4.浮点数
浮点数的尾数是原码小数，规格化的原码小数是 0.1xxxxxx 格式，因此 -1 位的 1 可以隐藏，实际运算时我们就把隐藏位放在第 0 位，也即 1.xxxxxxx,这样实际发生了左移，我们再调整阶码即可

`±0.1xxx... × 2^{E}` --->  `±1.xxx... × 2^{E-1}`


规格化浮点数   `尾数 * 基数^指数` 中，尾数在 [1 , 基数）

5.定点小数

在计算机中，当我们需要直接表示一个小数时，最直观的方式是使用**定点数**。其核心思想是：**预先约定小数点在数据中的位置，所有数字都按此约定进行解释。**

---
计算机的存储单元（如32位）本身只是一串`0`和`1`，没有物理的小数点。定点数表示法就是**人为地、固定地**约定这串二进制数中，哪几位是整数部分，哪几位是小数部分。

例如，我们约定一个16位的存储空间中：
*   **格式A（纯小数）**：最高位为符号位，小数点紧接着符号位之后。格式为 `S . X X X X X X X X X X X X X X X`。它能表示的范围是 `-1 ≤ 数值 < 1`。
*   **格式B（整数+小数）**：最高位为符号位，接下来8位为整数部分，最后7位为小数部分。格式为 `S I I I I I I I I . F F F F F F F`。例如，它能表示像 `-125.75` 这样的数。

**关键**：硬件电路本身并不知道这个约定，是编写程序的程序员和编译器“知道”这个约定，并在进行运算（如加减乘除）后，手动或由编译器自动调整结果的小数点位置。


表示 `+0.625` 和 `-0.625` (`0.625₁₀ = 0.101₂`)
*   **原码**：`+0.625` -> `0 101`； `-0.625` -> `1 101`
*   **反码**：`+0.625` -> `0 101`； `-0.625` -> `1 010` （数值位`101`取反为`010`）
*   **补码**：`+0.625` -> `0 101`； `-0.625` -> `1 011` （反码`1 010` + 1 = `1 011`）


## 指令系统
1.注意PC在什么时候自增，如果是取指令时，那么相对寻址时PC已经指向了下一条指令。目标地址范围不等于位移量D的表示范围！​ 必须根据编址方式、指令长度和位移量单位进行换算。注意偏移量相减是目标减当前，可能是个负数
只有当计算得到的目标地址恰好是某条指令的起始（首）字节地址时，这次跳转才是正确的。（也即地址值要是指令字节数的倍数）

当一条指令要分多次加载时，PC也要多次自增

2.扩展操作码：要保证指令没有歧义，从操作数占位多的指令开始编码，后续指令不能使用之前用过的操作码作为前缀。例如 2地址指令操作码4位，使用了 k2 位，那么对于后续指令前4位能使用的操作码只有 `2^4 - k2` 种，1地址指令操作码多6位，指令k1 条，那么前10位剩余操作码 `[(2^4 - k2)*2^6 - k1]` ,零地址操作码再多6位 `k0max = [(2^4 - k2)*2^6 - k1] * 2^6`

3.实现f = g + h - j;要用额外的寄存器存放临时运算结果，不能改变没有改变的变量

4.注意选择结构 是 `bne/beq` ,`else` 分支前要有 `j exit` 

5.访问a[i]:先计算目标的有效地址，再 lw 时偏移量为0即可，注意`目标地址 = arr + i * 4`

6.被调用者（callee）不是不能用保留寄存器，而是要负责恢复它，因此保存寄存器的值入栈（写入内存），以便在栈返回时重新lw回原来的reg；如果callee不使用保留reg,则无需入栈


7.调用的过程：
`jal`: 把返回地址（PC + 4）写入 `$ra` , jump 到 callee 

`callee`:
第一步，函数序言，计算本函数所需的栈空间，一次性移动 `$sp` 到目标位置；依次把 `$ra` `$fp` `（被使用的）$s` `其他数据` 写入栈内存；设置 `$fp` 的值为存储 `$ra` 的地址；执行自己的任务，从 `$a` 中获取参数，并将返回值写入 `$v`；把栈内存中保存的数据 lw 回对应 reg ；恢复帧指针（退栈）；最后 `jr $ra`

需要分配栈帧的通常有三种情况，只要满足其一，就需要栈帧：
`1` callee自己是“非叶子函数”（它还会调用其他函数，因此必须保存 $ra)

`2` callee需要使用一些被调用者保存寄存器（如 `$s0`-`$s7`），因此必须保存它们的原始值。

`3` callee的局部变量太多，无法全部放入寄存器，需要在栈上分配空间（如局部数组）。

8.“字”是计算机中处理、传输或编址的一个自然单位。这个“自然单位”的大小，是由该计算机的设计决定的，主要是其CPU的字长。一般 MDR位宽至少和字长一致，MAR位宽由内存单元数决定

9.注意区分 按字编址和按字节编址

10.注意地址转移范围要考虑主存总空间大小的关系，最终取交集
