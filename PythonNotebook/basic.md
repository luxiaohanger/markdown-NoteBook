## 变量、对象
从底层本质来看，Python 的一切设计都围绕着两个核心：**对象（Object）** 是内存中的实体，而 **变量（Variable）** 只是指向这些实体的“便利贴”。

### 1. 变量：它只是一个标签（Name）

在 C++ 或 Java 中，变量通常被视为一个“盒子”，有固定的类型和空间。但在 Python 中，**变量本身没有类型，也没有空间**。

* **本质**：变量是一个存储在 **命名空间（Namespace）** 中的**引用（Reference）/ 指针**。
* **动作**：当你写 `a = [1, 2]` 时，Python 并不是把列表放进 `a` 这个盒子里，而是先在内存里造出一个列表对象，然后把 `a` 这个标签贴在它上面。


### 2. 对象：内存中的实体

Python 中“万物皆对象”。每一个对象在内存中由三部分组成：

1.  **身份 (Identity)**：唯一的内存地址，通过 `id()` 查看。
2.  **类型 (Type)**：告诉 Python 这个对象能干什么（比如能加减还是能迭代），通过 `type()` 查看。
3.  **值 (Value)**：对象存储的具体数据。

#### 可变对象 vs 不可变对象
这是理解 Python 的关键：
* **不可变（Immutable）**：`int`, `float`, `str`, `tuple`。一旦创建，值就不能改。如果你修改 `x = 1; x = 2`，其实是把 `x` 从 `1` 的身上撕下来，贴到了新生成的 `2` 身上。
* **可变（Mutable）**：`list`, `dict`, `set`。对象内部的值可以被原地修改，而地址（标签指向的位置）保持不变。


### 3. 函数参数：共享传参 (Pass-by-object-reference)

关于 Python 是“值传递”还是“引用传递”的争论很多，但最准确的定义是 **共享传参（Call by sharing）**。

#### 本质过程
当你把一个变量传递给函数时，函数参数（局部变量）会变成该对象的**第二个标签**。

1.  **传递不可变对象**：
    如果你在函数内部修改它，由于对象不可变，局部变量会指向一个**新对象**。对外部原始变量没有影响。
    ```python
    def update(n):
        n = n + 1 # 这里的 n 指向了新生成的整数对象
    ```

2.  **传递可变对象**：
    由于局部变量和全局变量指向的是**同一个内存实体**，如果你在函数内通过 `append()` 等方法修改对象，外部也会感知到变化。
    ```python
    def update(L):
        L.append(4) # 原地修改，外部的列表也变了
    ```
