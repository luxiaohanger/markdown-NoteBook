# 驻留

Python 的驻留缓存机制是一种内存优化技术，它会**存储不可变对象的唯一副本**，以便在多个地方重复使用，从而**节省内存**并**加快比较速度**（通过地址比较而不是值比较）。

## 主要驻留对象

### 1. 小整数对象池
- **范围**：-5 到 256
- Python 在启动时预先创建这些整数对象
- 程序中所有使用这些整数的变量都指向同一个对象
```python
a = 100
b = 100
print(a is b)  # True
c = 257
d = 257
print(c is d)  # 在交互式环境中通常为False
```

### 2. 字符串驻留机制
Python 会自动驻留以下字符串：
- **标识符**：变量名、函数名、类名等
- **短字符串**：通常长度不超过 20 个字符，且仅包含字母、数字、下划线
- **空字符串**和**单字符**字符串
```python
# 自动驻留
s1 = "hello"
s2 = "hello"
print(s1 is s2)  # True

# 包含特殊字符的长字符串通常不驻留
s3 = "hello world!"
s4 = "hello world!"
print(s3 is s4)  # 可能为True（字面量相同），但不保证

# 动态创建的字符串不自动驻留
s5 = "hello"
s6 = "".join(['h', 'e', 'l', 'l', 'o'])
print(s5 is s6)  # False
```

### 3. 单例对象
- **`None`、`True`、`False`**：全局唯一
- **空元组**：`()` 是单例
- **空不可变对象**：如空`frozenset`等
```python
print(None is None)  # True
print(True is True)  # True
print(() is ())      # True
```

### 4. 代码对象中的常量
- 函数、类定义中的字面量可能会被驻留
- 编译时确定的常量

## 显式字符串驻留
可以使用 `sys.intern()` 函数强制驻留字符串，特别适用于处理大量可能重复的文本数据：
```python
import sys

# 动态创建大量相同字符串时
s1 = sys.intern("这是一个很长的重复字符串" * 10)
s2 = sys.intern("这是一个很长的重复字符串" * 10)
print(s1 is s2)  # True，节省内存

# 适用于文本处理、自然语言处理等场景
words = [sys.intern(word) for word in large_text_corpus]
```

## 驻留机制的特点
1. **不可变性**：只有不可变对象才能被安全驻留
2. **实现相关**：不同Python实现（CPython、PyPy等）的驻留策略可能不同
3. **编译时优化**：相同字面量在编译阶段可能被合并
4. **不保证性**：除了小整数和单例，其他对象的驻留不是语言保证的

## 使用建议
- **比较操作**：使用 `==` 进行值比较，不要依赖 `is` 比较（除非比较 `None`、`True`、`False`）
- **内存优化**：处理大量重复字符串时考虑使用 `sys.intern()`
- **理解而非依赖**：了解机制有助于调试和优化，但不应依赖其具体行为编写代码

驻留机制是Python实现细节的一部分，主要目的是优化性能，而不是语言规范要求。
